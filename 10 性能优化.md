# 二、启动速度优化

## 冷启动流程

```java
Launcher
   ↓
Zygote fork
   ↓
ActivityThread.main()
   ↓
Application.onCreate()
   ↓
Activity.onCreate()
   ↓
首帧绘制
```

---

## 常见问题

### Application初始化过多

错误：

```java
public class App extends Application {

    @Override
    public void onCreate() {

        initBugly();
        initIM();
        initDatabase();
        initRouter();
        initXXX();

    }
}
```

导致：

```text
Application耗时500ms+
```

---

## 优化方案

### 1 延迟初始化

```java
Handler().post(() -> {
    initBugly();
});
```

或者：

```java
Executors.newSingleThreadExecutor()
    .execute(() -> initDatabase());
```

---

### 2 Jetpack Startup

```java
public class BuglyInitializer
        implements Initializer<Bugly> {

}
```

按需初始化。

---

### 3 异步加载

```java
CompletableFuture.runAsync(() -> {
    initIM();
});
```

# 三、内存优化

---

## 内存泄漏

典型案例：

### Handler

```java
class MainActivity extends Activity {

    Handler handler = new Handler();

}
```

Handler持有Activity引用。

---

优化：

```java
static class MyHandler extends Handler {

    WeakReference<Activity> ref;

}
```

---

### 单例持有Activity

错误：

```java
public class Manager {

    private Activity activity;
}
```

改：

```java
private Context appContext;
```

---

### 匿名内部类

```java
new Thread() {

};
```

持有外部Activity引用。

---

## LeakCanary检测

```gradle
debugImplementation
'com.squareup.leakcanary:leakcanary-android'
```

自动检测泄漏。

# 四、卡顿优化（UI优化）

Android要求：

```text
16.6ms
完成一帧
```

否则：

```text
掉帧
Jank
```

---

## 原因

### 主线程耗时

错误：

```java
button.setOnClickListener(v -> {

    Thread.sleep(2000);

});
```

---

优化：

```java
Executors.newCachedThreadPool()
        .execute(...)
```

---

### 布局层级过深

错误：

```xml
LinearLayout
    LinearLayout
        RelativeLayout
            FrameLayout
```

10层以上。

---

优化：

```xml
ConstraintLayout
```

减少层级。

---

### RecyclerView优化

#### ViewHolder复用

```java
@Override
public int getItemViewType(int position) {
    return 0;
}
```

避免频繁创建View。

---

#### DiffUtil

```java
DiffUtil.calculateDiff()
```

局部刷新：

```java
notifyItemChanged()
```

避免：

```java
notifyDataSetChanged()
```

# 五、图片优化

图片最容易OOM。

---

## 压缩加载

Glide：

```java
Glide.with(this)
    .load(url)
    .override(300,300)
    .into(iv);
```

---

## RGB565

```java
RequestOptions options =
    new RequestOptions()
        .format(DecodeFormat.PREFER_RGB_565);
```

内存减半：

```text
ARGB_8888 4字节
RGB_565   2字节
```

---

## 图片缓存

Glide三级缓存：

```text
Active Resources

Memory Cache

Disk Cache
```

# 六、网络优化

---

## OkHttp连接池

```java
ConnectionPool
```

复用TCP连接。

避免：

```text
每次请求重新三次握手
```

---

## GZIP压缩

```http
Accept-Encoding:gzip
```

减少流量。

---

## HTTP缓存

```java
Cache cache =
new Cache(dir, 20 * 1024 * 1024);
```

减少重复请求。

# 七、数据库优化

---

## 建索引

错误：

```sql
select * from user
where phone='123'
```

无索引。

---

优化：

```sql
create index idx_phone
on user(phone);
```

---

## 批量插入

错误：

```java
for(...) {
 insert();
}
```

---

优化：

```java
beginTransaction();
```

性能提升数十倍。

# 九、电量优化

---

## 避免频繁WakeLock

错误：

```java
wakeLock.acquire();
```

忘记释放。

---

优化：

```java
try {

} finally {

    wakeLock.release();

}
```

---

## 使用WorkManager

代替：

```java
while(true){
   请求服务器
}
```

使用：

```java
WorkManager
```

系统统一调度。

# 面试回答模板

如果面试官问：

**"你做过哪些性能优化？"**

可以这样回答：

> 在项目中我主要从启动优化、内存优化、卡顿优化、网络优化和包体积优化几个方面进行性能治理。
> 
> - 启动阶段通过异步初始化、延迟加载和 Jetpack Startup 将冷启动时间缩短约30%；
>     
> - 使用 LeakCanary 检测内存泄漏，重点解决 Handler、单例、匿名内部类导致的泄漏问题；
>     
> - 利用 Systrace 和 Perfetto 分析卡顿，优化 RecyclerView、减少布局层级、避免主线程耗时操作；
>     
> - 网络层通过 OkHttp 连接池、缓存和 GZIP 压缩减少请求耗时；
>     
> - APK方面通过 R8 混淆、资源压缩和 ABI 过滤减少安装包体积；
>     
> - 最终应用启动时间从约 2.5 秒优化到 1.6 秒，内存占用下降约 20%，页面掉帧率明显降低。
>     

这个回答是高级 Android 工程师面试中比较完整且有项目深度的回答方式。
