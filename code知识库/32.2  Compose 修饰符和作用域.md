## 修饰符基础

### 什么是修饰符（Modifier）

修饰符是 Compose 中用来装饰或增强可组合项的对象，通过链式调用可以组合多个修饰符的功能。

```kotlin
Text(
    text = "Hello",
    modifier = Modifier
        .size(100.dp)
        .background(Color.Red)
        .padding(16.dp)
        .clickable { /* ... */ }
)
```

### 修饰符的执行顺序（从右到左）

修饰符是从右向左应用的，最右边的修饰符首先被应用：

```kotlin
Modifier
    .padding(16.dp)      // 4. 最后应用，影响最终外观
    .background(Color.Red) // 3. 其次
    .size(100.dp)        // 2. 其次
    .clickable { }       // 1. 首先应用
```

## 布局修饰符

### 1. padding() - 内边距

**作用**：在元素内部添加空间

```kotlin
// 所有边都一样
Modifier.padding(16.dp)

// 水平和竖直分别指定
Modifier.padding(horizontal = 16.dp, vertical = 8.dp)

// 单独指定
Modifier.padding(start = 16.dp, top = 8.dp, end = 16.dp, bottom = 8.dp)

// 使用 PaddingValues
Modifier.padding(PaddingValues(16.dp))
```

**约束条件**：

- 值必须是正数，不能是负数
- 过大的 padding 可能导致内容溢出

### 2. margin() 等同概念 - offset()

**作用**：相对于当前位置的偏移

```kotlin
Modifier.offset(x = 10.dp, y = 20.dp)
```

**约束条件**：

- offset() 不改变布局尺寸，只是视觉偏移
- 不会导致重新布局，性能较好

### 3. fillMaxWidth() / fillMaxHeight() / fillMaxSize()

**作用**：填充最大可用空间

```kotlin
// 填充宽度
Modifier.fillMaxWidth()

// 填充百分比
Modifier.fillMaxWidth(0.8f)  // 80% 宽度

// 填充高度
Modifier.fillMaxHeight()
Modifier.fillMaxHeight(0.5f)

// 同时填充宽高
Modifier.fillMaxSize()
Modifier.fillMaxSize(0.9f)
```

**约束条件**：

- 百分比参数范围：0f 到 1f（浮点数）
- 只在父布局有约束的情况下有效
- 在 Column 中使用 fillMaxHeight() 必须配合 Modifier.weight()

### 4. width() / height() / size()

**作用**：设置固定尺寸

```kotlin
// 固定宽度
Modifier.width(200.dp)

// 固定高度
Modifier.height(100.dp)

// 固定宽高
Modifier.size(100.dp)

// 分别指定
Modifier.size(width = 100.dp, height = 50.dp)
```

**约束条件**：

- 值必须是非负数
- 如果内容超出设定尺寸，会被裁剪
- 与 fillMaxWidth() 冲突时，后者优先级更高

### 5. wrapContentWidth() / wrapContentHeight() / wrapContentSize()

**作用**：内容包裹（自适应内容大小）

```kotlin
Modifier.wrapContentWidth()
Modifier.wrapContentWidth(Alignment.Center)  // 对齐方式

Modifier.wrapContentHeight()
Modifier.wrapContentHeight(Alignment.Center)

Modifier.wrapContentSize()
Modifier.wrapContentSize(Alignment.Center)
```

**约束条件**：

- 覆盖了尺寸约束，自动调整大小以适应内容
- 通常用于特殊布局场景

### 6. weight()

**作用**：在 Row/Column 中分配权重，分享剩余空间

```kotlin
Row {
    Text("A", modifier = Modifier.weight(1f))  // 占 50%
    Text("B", modifier = Modifier.weight(1f))  // 占 50%
}

Row {
    Text("A", modifier = Modifier.weight(2f))  // 占 66.7%
    Text("B", modifier = Modifier.weight(1f))  // 占 33.3%
}
```

**约束条件**：

- 权重必须大于 0
- 权重是相对的，不是绝对值
- 只在 RowScope 和 ColumnScope 中有效
- fill 参数（默认 true）决定是否填充整个方向

## 尺寸修饰符

### 1. sizeIn()

**作用**：设置尺寸约束范围

```kotlin
Modifier.sizeIn(
    minWidth = 50.dp,
    maxWidth = 200.dp,
    minHeight = 40.dp,
    maxHeight = 100.dp
)
```

**约束条件**：

- min 值必须 ≤ max 值
- 如果不指定 min，默认为 0.dp
- 如果不指定 max，默认为 Infinity

### 2. widthIn() / heightIn()

**作用**：仅限制单一方向

```kotlin
Modifier.widthIn(min = 50.dp, max = 200.dp)
Modifier.heightIn(min = 40.dp, max = 100.dp)
```

### 3. defaultMinSize()

**作用**：设置默认最小尺寸

```kotlin
Modifier.defaultMinSize(minWidth = 48.dp, minHeight = 48.dp)
```

**约束条件**：

- 常用于按钮等组件的最小点击区域
- Material Design 通常要求最小 48.dp

### 4. aspectRatio()

**作用**：维持宽高比

```kotlin
// 1:1 的正方形
Modifier.aspectRatio(1f)

// 16:9 的矩形
Modifier.aspectRatio(16f / 9f)

// 不固定一个维度
Modifier.size(200.dp).aspectRatio(1f)  // 200x200
Modifier.fillMaxWidth().aspectRatio(16f / 9f)  // 宽充满，高自动计算
```

**约束条件**：

- 宽高比必须 > 0
- 必须至少一个维度是固定的或受约束的

## 外观修饰符

### 1. background()

**作用**：设置背景

```kotlin
// 简单颜色
Modifier.background(Color.Red)

// 带圆角
Modifier.background(Color.Red, shape = RoundedCornerShape(8.dp))

// 自定义 Shape
Modifier.background(
    brush = SolidColorBrush(Color.Red),
    shape = CircleShape
)

// 使用 Brush（渐变）
Modifier.background(
    brush = LinearGradientBrush(
        colors = listOf(Color.Red, Color.Blue),
        start = Offset(0f, 0f),
        end = Offset(100f, 100f)
    )
)
```

**约束条件**：

- shape 参数仅当使用 brush 时才能定义
- 背景会被应用到 padding 外部（在 background() 调用之前的修饰符）

### 2. border()

**作用**：添加边框

```kotlin
Modifier.border(
    width = 2.dp,
    color = Color.Black,
    shape = RoundedCornerShape(8.dp)
)

// 使用 Brush
Modifier.border(
    width = 2.dp,
    brush = SolidColorBrush(Color.Black),
    shape = CircleShape
)
```

**约束条件**：

- 边框宽度必须非负
- 边框绘制在内容外部

### 3. clip() 和 clipToBounds()

**作用**：裁剪内容

```kotlin
// 按照指定的 Shape 裁剪
Modifier.clip(RoundedCornerShape(8.dp))
Modifier.clip(CircleShape)

// 裁剪到矩形边界
Modifier.clipToBounds()
```

**约束条件**：

- 裁剪会影响性能，应避免过度使用
- 裁剪后的内容不可见，不占用空间

### 4. shadow()

**作用**：添加阴影

```kotlin
Modifier.shadow(
    elevation = 4.dp,
    shape = RoundedCornerShape(8.dp),
    clip = true  // 是否裁剪到 shape
)
```

**约束条件**：

- elevation 必须非负
- clip = true 时，内容会被裁剪到 shape
- 在 Android 层面实际通过 canvas 绘制，性能开销较大

### 5. alpha()

**作用**：设置透明度

```kotlin
Modifier.alpha(0.5f)  // 50% 透明

Modifier.alpha(0f)    // 完全透明（不占用交互空间）
Modifier.alpha(1f)    // 完全不透明
```

**约束条件**：

- 透明度范围：0f 到 1f
- alpha() 不影响组件是否能接收点击事件（与 visibility 不同）

### 6. drawWithContent() / drawBehind()

**作用**：自定义绘制

```kotlin
Modifier.drawBehind {
    drawCircle(color = Color.Red, radius = 50f)
}

Modifier.drawWithContent {
    drawCircle(color = Color.Blue)
    drawContent()  // 绘制组件自身内容
    drawCircle(color = Color.Green)
}
```

## 交互修饰符

### 1. clickable()

**作用**：处理点击事件

```kotlin
Modifier.clickable {
    println("Clicked!")
}

Modifier.clickable(
    enabled = true,
    onClickLabel = "Activate button",
    role = Role.Button,
    onClick = { /* ... */ }
)
```

**约束条件**：

- 如果 enabled = false，点击不响应，且无视觉反馈
- 必须设置最小尺寸 (通常 48.dp) 满足 Material 规范

### 2. pointerInput()

**作用**：处理底层指针事件

```kotlin
Modifier.pointerInput(Unit) {
    detectTapGestures(
        onTap = { offset ->
            println("Tapped at $offset")
        }
    )
}

Modifier.pointerInput(Unit) {
    detectDragGestures(
        onDrag = { change, offset ->
            change.consume()
            println("Dragged by $offset")
        }
    )
}
```

**约束条件**：

- 第一个参数是 key，改变时会重启手势检测
- 必须调用 consume() 防止事件继续传递
- 复杂手势可能与其他修饰符冲突

### 3. draggable()

**作用**：支持拖拽

```kotlin
var offset by remember { mutableStateOf(0f) }

Modifier.draggable(
    state = rememberDraggableState { delta ->
        offset += delta
    },
    orientation = Orientation.Horizontal
)
```

**约束条件**：

- 只支持单一方向
- 需要配合 State 管理偏移

### 4. scrollable()

**作用**：使组件可滚动

```kotlin
var offset by remember { mutableStateOf(0f) }

Modifier.scrollable(
    state = rememberScrollableState { delta ->
        offset += delta
        delta
    },
    orientation = Orientation.Vertical
)
```

### 5. focusable() 和 focusModifier()

**作用**：处理焦点

```kotlin
Modifier.focusable()

val focusRequester = FocusRequester()
Modifier
    .focusRequester(focusRequester)
    .onFocusChanged { focusState ->
        if (focusState.isFocused) {
            println("Focused!")
        }
    }
```

## 变换修饰符

### 1. scale()

**作用**：缩放

```kotlin
// 等比缩放
Modifier.scale(2f)

// 分别缩放
Modifier.scale(scaleX = 2f, scaleY = 1f)
```

**约束条件**：

- 缩放不改变布局尺寸（只是视觉效果）
- 如果缩放后超出边界，会被裁剪

### 2. rotate()

**作用**：旋转

```kotlin
Modifier.rotate(45f)  // 45 度
```

**约束条件**：

- 度数范围通常 0-360，但可以超出
- 旋转不改变布局尺寸

### 3. graphicsLayer()

**作用**：综合图形变换（性能优化）

```kotlin
Modifier.graphicsLayer {
    scaleX = 1.5f
    scaleY = 1.5f
    rotationZ = 45f
    translationX = 10.dp.toPx()
    translationY = 20.dp.toPx()
    alpha = 0.8f
    shadowElevation = 4.dp.toPx()
    clip = true
}
```

**约束条件**：

- 比单独的 scale()、rotate() 性能更好
- 变换在 RenderNode 层面进行
- 修改某些属性需要手动触发重绘

### 4. transform()

**作用**：矩阵变换

```kotlin
Modifier.transform { originalSize ->
    IntOffset(x = originalSize.width / 2, y = originalSize.height / 2)
}
```

## 作用域详解

### 1. RowScope

**可用的独特修饰符**：`weight()`、`align()`

```kotlin
Row {
    Text(
        text = "Item 1",
        modifier = Modifier
            .weight(1f)  // 仅在 RowScope 有效
            .align(Alignment.Top)  // 垂直对齐
    )
    
    Text(
        text = "Item 2",
        modifier = Modifier
            .weight(2f)
            .align(Alignment.Bottom)
    )
}
```

**RowScope.align() 的对齐选项**：

- `Alignment.Top`
- `Alignment.CenterVertically`
- `Alignment.Bottom`

### 2. ColumnScope

**可用的独特修饰符**：`weight()`、`align()`

```kotlin
Column {
    Text(
        text = "Header",
        modifier = Modifier
            .weight(1f)  // 仅在 ColumnScope 有效
            .align(Alignment.Start)  // 水平对齐
    )
    
    Spacer(modifier = Modifier.weight(1f))
    
    Text(
        text = "Footer",
        modifier = Modifier
            .weight(1f)
            .align(Alignment.End)
    )
}
```

**ColumnScope.align() 的对齐选项**：

- `Alignment.Start`
- `Alignment.CenterHorizontally`
- `Alignment.End`

### 3. BoxScope

**可用的独特修饰符**：`align()`、`matchParentSize()`

```kotlin
Box {
    // 背景
    Box(
        modifier = Modifier
            .matchParentSize()
            .background(Color.Gray)
    )
    
    // 前景文字，对齐到中心
    Text(
        text = "Center",
        modifier = Modifier.align(Alignment.Center)
    )
    
    // 对齐到右下
    Text(
        text = "Bottom-End",
        modifier = Modifier.align(Alignment.BottomEnd)
    )
}
```

**BoxScope.align() 的对齐选项**：

- `Alignment.TopStart`、`Alignment.TopCenter`、`Alignment.TopEnd`
- `Alignment.CenterStart`、`Alignment.Center`、`Alignment.CenterEnd`
- `Alignment.BottomStart`、`Alignment.BottomCenter`、`Alignment.BottomEnd`

**matchParentSize()**：

- 使 Box 内的子元素占满父 Box 的全部空间
- 通常用于背景层

### 4. LazyListScope / LazyGridScope

**可用的独特修饰符**：`animateItem()` (Compose 1.6+)

```kotlin
LazyColumn {
    items(items.size) { index ->
        Text(
            text = items[index],
            modifier = Modifier.animateItem()  // 仅在 LazyListScope 有效
        )
    }
}
```

**animateItem() 的约束**：

- 仅支持列表项的出入动画
- 需要稳定的项目 key

### 5. CompositionLocal 作用域

**作用**：通过隐式作用域传递数据

```kotlin
val LocalCustomData = compositionLocalOf { "default" }

@Composable
fun ParentComposable() {
    CompositionLocalProvider(
        LocalCustomData provides "custom value"
    ) {
        ChildComposable()  // 可以访问 LocalCustomData.current
    }
}

@Composable
fun ChildComposable() {
    Text(LocalCustomData.current)
}
```

**约束条件**：

- CompositionLocal 不能跨越 BackHandler、Dialog 等边界
- 过度使用会导致可读性下降

## 约束条件总结

### 尺寸约束规则

1. **约束层级**（从高到低）：
    
    - 显式尺寸修饰符 (width, height, size)
    - fillMaxWidth() / fillMaxHeight()
    - weight()
    - wrapContent()
    - 组件默认大小
2. **冲突解决**：
    
    ```kotlin
    // width(100.dp) 优先级更高
    Modifier
        .width(100.dp)
        .fillMaxWidth()
    ```
    
3. **weight() 的约束**：
    
    - 只在 Row/Column 中有效
    - 不能与固定尺寸混用
    - 权重百分比 = 自身权重 / 所有权重之和

### 性能约束

1. **避免频繁的状态变化**：
    
    ```kotlin
    // ❌ 不好：每次状态变化都重组
    var offset by remember { mutableStateOf(0f) }
    Modifier.offset(x = offset.dp)
    
    // ✅ 好：使用 graphicsLayer
    Modifier.graphicsLayer {
        translationX = offset
    }
    ```
    
2. **shadow() 和 clip() 性能开销大**：
    
    ```kotlin
    // 尽量使用简单的修饰符
    // 避免在复杂列表中使用 shadow()
    ```
    
3. **避免在 LazyColumn 中使用重型修饰符**：
    
    ```kotlin
    LazyColumn {
        items(1000) { index ->
            // ❌ 不要在这里使用 shadow() 或复杂的 graphicsLayer
            // ✅ 使用轻量级修饰符
        }
    }
    ```
    

### 空间约束

1. **内容溢出处理**：
    
    ```kotlin
    // 自动截断
    Modifier
        .size(100.dp)
        .clip(RoundedCornerShape(8.dp))
    
    // 自动换行
    Text(
        text = "Long text",
        modifier = Modifier.fillMaxWidth(),
        maxLines = 1,
        overflow = TextOverflow.Ellipsis
    )
    ```
    
2. **负 padding 不被允许**：
    
    ```kotlin
    // ❌ 错误
    Modifier.padding(-16.dp)
    
    // ✅ 正确：使用 offset
    Modifier.offset(x = (-16).dp)
    ```
    

### 交互约束

1. **最小点击区域**：
    
    ```kotlin
    // Material 规范要求最少 48.dp
    Modifier
        .clickable { }
        .defaultMinSize(minWidth = 48.dp, minHeight = 48.dp)
    ```
    
2. **焦点和选择状态**：
    
    ```kotlin
    // 互斥：一个时间只能一个获得焦点
    Modifier.focusable()
    ```
    

### 动画约束

1. **animateDpAsState**：
    
    ```kotlin
    val offsetDp by animateDpAsState(if (isExpanded) 100.dp else 0.dp)
    Modifier.offset(y = offsetDp)
    ```
    
2. **Animatable（低级 API）**：
    
    ```kotlin
    val offset = remember { Animatable(0f) }
    LaunchedEffect(Unit) {
        offset.animateTo(100f)
    }
    Modifier.graphicsLayer {
        translationX = offset.value
    }
    ```

## 实际应用示例

### 完整的按钮实现

```kotlin
@Composable
fun CustomButton(
    text: String,
    onClick: () -> Unit,
    modifier: Modifier = Modifier,
    enabled: Boolean = true
) {
    var isPressed by remember { mutableStateOf(false) }
    
    Box(
        modifier = modifier
            .defaultMinSize(minWidth = 48.dp, minHeight = 48.dp)
            .background(
                color = if (enabled) Color.Blue else Color.Gray,
                shape = RoundedCornerShape(4.dp)
            )
            .border(
                width = 1.dp,
                color = Color.DarkGray,
                shape = RoundedCornerShape(4.dp)
            )
            .clickable(enabled = enabled) {
                onClick()
            }
            .graphicsLayer {
                scaleX = if (isPressed) 0.95f else 1f
                scaleY = if (isPressed) 0.95f else 1f
            }
            .pointerInput(Unit) {
                awaitEachGesture {
                    awaitFirstDown()
                    isPressed = true
                    
                    waitForUpOrCancellation()
                    isPressed = false
                }
            },
        contentAlignment = Alignment.Center
    ) {
        Text(
            text = text,
            color = Color.White,
            modifier = Modifier.padding(horizontal = 16.dp, vertical = 12.dp)
        )
    }
}
```

### 流布局（Flow）

```kotlin
@Composable
fun ResponsiveFlow(
    modifier: Modifier = Modifier,
    content: @Composable () -> Unit
) {
    var totalWidth by remember { mutableStateOf(0) }
    
    Layout(
        content = content,
        modifier = modifier,
        measurePolicy = { measurables, constraints ->
            val placeables = measurables.map { 
                it.measure(constraints.copy(maxWidth = Constraints.Infinity))
            }
            
            var currentX = 0
            var currentY = 0
            var maxHeight = 0
            
            val positions = mutableListOf<Pair<Int, Int>>()
            
            placeables.forEach { placeable ->
                if (currentX + placeable.width > constraints.maxWidth) {
                    currentX = 0
                    currentY += maxHeight
                    maxHeight = 0
                }
                
                positions.add(currentX to currentY)
                currentX += placeable.width
                maxHeight = maxOf(maxHeight, placeable.height)
            }
            
            layout(constraints.maxWidth, currentY + maxHeight) {
                placeables.forEachIndexed { index, placeable ->
                    val (x, y) = positions[index]
                    placeable.place(x, y)
                }
            }
        }
    )
}
```

## 常见陷阱和解决方案

|问题|原因|解决方案|
|---|---|---|
|fillMaxWidth() 不起作用|父布局没有约束|明确指定父布局宽度|
|weight() 无法均分|修饰符顺序错误|确保在 layout 修饰符之前|
|padding() 被背景覆盖|修饰符顺序|将 background() 放在 padding() 之前|
|点击区域太小|没有设置最小尺寸|添加 defaultMinSize()|
|性能下降|过度使用 shadow()、clip()|在 LazyColumn 中避免使用|
|无法响应点击|alpha(0f) 仍会消耗事件|使用 visible = false 或 enabled = false|
