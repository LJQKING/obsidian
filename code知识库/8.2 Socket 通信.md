# Android Socket 通信详解（面试 + 实战）

Socket 是网络通信的基础，Android 中 HTTP、WebSocket、MQTT 等协议底层都依赖 Socket。

---

# 一、Socket 是什么

Socket（套接字）是：

> 应用层与 TCP/IP 协议栈之间的接口

可以理解为：

```text
App
 ↓
Socket
 ↓
TCP
 ↓
IP
 ↓
网卡
```

Android客户端：

```java
Socket socket = new Socket(ip, port);
```

服务器：

```java
ServerSocket serverSocket = new ServerSocket(port);
```

建立连接后：

```text
Client Socket
        ↕
      TCP
        ↕
Server Socket
```

---

# 二、Socket 通信流程

## 客户端

```java
Socket
```

连接服务端：

```java
Socket socket =
        new Socket("192.168.1.100",8888);
```

获取输出流：

```java
OutputStream os =
        socket.getOutputStream();
```

发送数据：

```java
os.write("hello".getBytes());
```

获取输入流：

```java
InputStream is =
        socket.getInputStream();
```

接收数据：

```java
byte[] buffer = new byte[1024];
int len = is.read(buffer);
```

---

## 服务端

监听端口：

```java
ServerSocket serverSocket =
        new ServerSocket(8888);
```

等待客户端：

```java
Socket client =
        serverSocket.accept();
```

获取流：

```java
InputStream is =
        client.getInputStream();

OutputStream os =
        client.getOutputStream();
```

# 三、Android Socket 示例

## Client

```java
public class SocketClient {

    private Socket socket;

    public void connect() {

        new Thread(() -> {

            try {

                socket =
                        new Socket(
                                "192.168.1.100",
                                8888);

                OutputStream os =
                        socket.getOutputStream();

                os.write("Hello Server"
                        .getBytes());

            } catch (Exception e) {
                e.printStackTrace();
            }

        }).start();
    }
}
```

---

## Server

```java
public class SocketServer {

    public void start() {

        new Thread(() -> {

            try {

                ServerSocket server =
                        new ServerSocket(8888);

                while (true) {

                    Socket client =
                            server.accept();

                    InputStream is =
                            client.getInputStream();

                    byte[] buffer =
                            new byte[1024];

                    int len =
                            is.read(buffer);

                    String msg =
                            new String(
                                    buffer,
                                    0,
                                    len);

                    System.out.println(msg);
                }

            } catch (Exception e) {
                e.printStackTrace();
            }

        }).start();
    }
}
```

# 四、TCP 三次握手

建立 Socket 连接时：

```text
Client                     Server

SYN -------------------->

     <------------------ SYN+ACK

ACK -------------------->
```

### 第一次

客户端：

```text
我要连接
```

发送：

```text
SYN
```

---

### 第二次

服务端：

```text
收到请求
```

返回：

```text
SYN + ACK
```

---

### 第三次

客户端：

```text
确认收到
```

返回：

```text
ACK
```

连接建立成功。

---

# 五、TCP 四次挥手

断开连接：

```text
Client                     Server

FIN -------------------->

      <------------------ ACK

      <------------------ FIN

ACK -------------------->
```

原因：

TCP 全双工。

双方发送和接收要分别关闭。

# 八、粘包和拆包问题

TCP：

```text
Hello
World
```

可能收到：

```text
HelloWorld
```

或者：

```text
Hel
lo
Wor
ld
```

因为 TCP 是：

```text
字节流
```

没有消息边界。

---

## 解决方案1：固定长度

```text
1024字节
```

缺点：

浪费带宽。

---

## 解决方案2：分隔符

```text
Hello\n
World\n
```

读取：

```java
BufferedReader.readLine()
```

---

## 解决方案3：消息头长度（推荐）

协议：

```text
4字节长度 + 数据
```

例如：

```text
0005Hello
```

读取：

```java
DataInputStream
```

先读长度：

```java
int len = input.readInt();
```

再读数据：

```java
byte[] data = new byte[len];
input.readFully(data);
```

工业项目最常见。

# 十、Android Socket + Handler

接收线程：

```java
new Thread(() -> {

    while (true){

        int len = is.read(buffer);

        Message msg =
                handler.obtainMessage();

        msg.obj =
                Arrays.copyOf(
                        buffer,
                        len);

        handler.sendMessage(msg);
    }

}).start();
```

UI线程：

```java
Handler handler =
        new Handler(
                Looper.getMainLooper()) {

    @Override
    public void handleMessage(
            Message msg) {

        byte[] data =
                (byte[]) msg.obj;

        tv.setText(
                new String(data));
    }
};
```

# Android Socket 面试高频问题

### Socket 和 HTTP 区别？

|Socket|HTTP|
|---|---|
|传输层|应用层协议|
|长连接|默认短连接|
|实时性高|较低|
|自定义协议|固定协议|

---

### TCP 和 UDP 区别？

|TCP|UDP|
|---|---|
|可靠|不可靠|
|有连接|无连接|
|慢|快|
|有序|无序|

---

### TCP 为什么会粘包？

因为 TCP 是字节流协议，没有消息边界。

---

### Socket 为什么需要心跳？

检测连接是否存活，及时重连。

---

### Android 设备通信项目中怎么设计 Socket？

推荐工业级协议：

```text
帧头(2)
长度(2)
命令(1)
Payload(N)
CRC(2)
```

配合：

- 长连接
    
- 心跳机制
    
- 自动重连
    
- 发送队列
    
- ACK确认
    
- 超时重发
    
- CRC校验
    

这是蓝牙设备、充电桩、物联网设备、智能硬件项目中最常见的 Socket 架构。
