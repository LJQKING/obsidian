在 Kotlin 中，`var` 和 `val` 是定义变量最基本的两个关键字，它们最大的区别是**是否允许重新赋值**。

|关键字|全称|是否可重新赋值|类似 Java|
|---|---|---|---|
|`var`|Variable|✅ 可以|普通变量|
|`val`|Value|❌ 不可以|`final` 变量|

---
## 1. var（可变变量）

使用 `var` 定义的变量可以多次修改。

```kotlin
var age = 20

age = 21
age = 22

println(age) // 22
```

也可以指定类型：

```kotlin
var name: String = "Tom"

name = "Jack"
```

---

## 2. val（只读变量）

`val` 只能赋值一次，之后不能再重新赋值。

```kotlin
val age = 20

// 编译错误
age = 21
```

但是可以这样：

```kotlin
val name: String = "Tom"
```

不能再：

```kotlin
name = "Jack"   // 编译失败
```

---

## 3. val 并不是完全不可变

这是面试非常喜欢考的一点。

如果 `val` 保存的是**对象引用**，那么**引用不能变，但对象内容可以变**。

例如：

```kotlin
val list = mutableListOf(1, 2, 3)

list.add(4)

println(list) // [1, 2, 3, 4]
```

这里：

- `list` 这个引用不能改成其他对象
    
- 但 `MutableList` 内部的数据可以修改
    

下面这样才会报错：

```kotlin
val list = mutableListOf(1, 2)

list = mutableListOf(3, 4) // 编译错误
```

---

## 4. 真正不可变

如果想真正不可变，应使用只读集合：

```kotlin
val list = listOf(1, 2, 3)

// list.add(4) // 编译错误
```

---

## 5. Java 对比

Kotlin：

```kotlin
var age = 20
val name = "Tom"
```

相当于 Java：

```java
int age = 20;

final String name = "Tom";
```

---

## 6. 为什么 Kotlin 推荐使用 val？

官方建议：

> **能用 `val` 就不要用 `var`。**

原因：

- 代码更安全，不容易被意外修改
    
- 更符合函数式编程思想
    
- 更容易推断和维护
    
- 更适合多线程，减少共享可变状态带来的问题
    

---

## 7. 面试高频问题

**Q：`val` 是常量吗？**

不是。

```kotlin
val time = System.currentTimeMillis()
```

`time` 的值是在运行时确定的，只是**不能再次赋值**。

真正的编译期常量需要使用：

```kotlin
const val PI = 3.1415926
```

`const val` 必须满足：

- 位于顶层或 `object` 中
    
- 类型只能是基本类型或 `String`
    
- 初始化值必须是编译期常量
    

---

### 一句话总结

- **`var`**：变量（Variable），**可以重新赋值**。
    
- **`val`**：只读变量（Value），**不能重新赋值**，但如果引用的是可变对象，对象内容仍然可以修改。
    
- **优先使用 `val`，只有确实需要改变引用时才使用 `var`。**

“可控类型”这个词比较宽泛，不同领域含义不同。常见有以下几种：

### 2. Kotlin 的可空类型（Nullable Type）

如果你想问的是 **可空类型**（很多人会写成"可控类型"），那么就是：

```kotlin
var name: String = "Tom"      // 不可空
var age: String? = null       // 可空
```

可空类型需要安全调用：

```kotlin
age?.length
```

或者

```kotlin
age!!.length
```

---

### 3. Java/Kotlin 引用类型和值类型

有时也会说对象是否"可控制"、"可变"。

#### 不可变类型（Immutable）

```kotlin
String
Int
Long
Boolean
```

创建后值不能修改。

#### 可变类型（Mutable）

```kotlin
MutableList
MutableMap
StringBuilder
```

例如：

```kotlin
val list = mutableListOf(1,2,3)
list.add(4)
```

---

### 4. Compose 中的状态可控（Controlled State）

例如：

```kotlin
TextField(
    value = text,
    onValueChange = {
        text = it
    }
)
```

这里 `TextField` 就属于**可控组件（Controlled Component）**。

相对应还有：

```kotlin
remember { mutableStateOf("") }
```

管理状态。

---

### 5. UI 控件类型（Control）

如果是 Android UI，可以理解为控件类型，例如：

- TextView
    
- EditText
    
- Button
    
- ImageView
    
- RecyclerView
    
- Compose 的 Button、TextField、LazyColumn 等
