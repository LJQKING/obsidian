# 六、外部拦截法

面试最爱问。

父容器决定。

---

父布局：

```java
@Override
public boolean onInterceptTouchEvent(MotionEvent ev) {

    switch (ev.getAction()) {

        case MotionEvent.ACTION_DOWN:
            mLastY = ev.getY();
            break;

        case MotionEvent.ACTION_MOVE:

            float dy = ev.getY() - mLastY;

            if (Math.abs(dy) > 10) {
                return true;
            }

            break;
    }

    return super.onInterceptTouchEvent(ev);
}
```

逻辑：

```text
父容器分析方向
决定是否拦截
```

---

# 七、内部拦截法

子View决定。

---

核心API：

```java
requestDisallowInterceptTouchEvent()
```

源码：

```java
ViewGroup.java
```

```java
public void requestDisallowInterceptTouchEvent(
        boolean disallowIntercept) {

    if (disallowIntercept ==
            ((mGroupFlags &
             FLAG_DISALLOW_INTERCEPT) != 0)) {
        return;
    }

    if (disallowIntercept) {
        mGroupFlags |= FLAG_DISALLOW_INTERCEPT;
    } else {
        mGroupFlags &= ~FLAG_DISALLOW_INTERCEPT;
    }

    if (mParent != null) {
        mParent.requestDisallowInterceptTouchEvent(
                disallowIntercept);
    }
}
```

作用：

```text
通知父容器：
不要拦截我的事件
```

---

RecyclerView中：

```java
@Override
public boolean dispatchTouchEvent(MotionEvent ev) {

    getParent()
        .requestDisallowInterceptTouchEvent(true);

    return super.dispatchTouchEvent(ev);
}
```

## 为什么ACTION_DOWN一般不能拦截？

因为：

```text

UP

全部给父容器
```

```java
mFirstTouchTarget
```

## requestDisallowInterceptTouchEvent为什么对DOWN无效？

源码：

```java
ACTION_DOWN
```

时：

```java
resetTouchState();
```

↓

```java
FLAG_DISALLOW_INTERCEPT
```

被重置。

因此：

```text
DOWN阶段父容器仍然能拦截
```

# 十四、面试标准答案（3分钟版）

当面试官问：

> Android滑动冲突如何解决？

可以直接回答：

```text
Android滑动冲突本质是父View和子View同时希望消费MOVE事件。

事件分发核心流程是：

dispatchTouchEvent
→ onInterceptTouchEvent
→ onTouchEvent

解决方案主要有三种：

1. 外部拦截法
父容器重写onInterceptTouchEvent，
根据dx、dy判断滑动方向决定是否拦截。

2. 内部拦截法
子View通过
requestDisallowInterceptTouchEvent(true)
通知父容器不要拦截事件。

3. NestedScrolling机制
RecyclerView、CoordinatorLayout、
AppBarLayout等通过NestedScroll实现父子协同消费滑动距离。

源码层面核心位于ViewGroup.dispatchTouchEvent、
onInterceptTouchEvent以及NestedScrollingChildHelper和ViewParentCompat调用链。
```

这一套回答基本可以覆盖 **阿里、腾讯、美团、字节、理想、小鹏、蔚来、华为车载AAOS** Android高级岗位关于滑动冲突的源码级追问。
