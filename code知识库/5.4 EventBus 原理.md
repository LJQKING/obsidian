很多面试官会问：

> EventBus 和 Handler 有什么关系？

> EventBus 为什么性能比广播高？

# 一、什么是 EventBus

EventBus 是一个发布-订阅（Publish/Subscribe）框架。

核心思想：

```text
发送者(Post)
      │
      ▼
   EventBus
      │
      ▼
订阅者(Subscriber)
```

发送方：

```java
EventBus.getDefault().post(new UserEvent());
```

接收方：

```java
@Subscribe(threadMode = ThreadMode.MAIN)
public void onUserEvent(UserEvent event) {
    tv.setText("收到消息");
}
```

发送者和接收者完全解耦。

---

# 二、为什么很多项目用 EventBus + Handler

例如：

### 错误写法

```java
@Subscribe(threadMode = ThreadMode.MAIN)
public void onEvent(BigDataEvent event) {
    parseJson();      // 耗时
    queryDB();        // 耗时
    updateUI();
}
```

因为：

```text
ThreadMode.MAIN
        ↓
主线程
        ↓
耗时操作
        ↓
掉帧
ANR
```

---

### 正确方式

```java
@Subscribe(threadMode = ThreadMode.BACKGROUND)
public void onEvent(BigDataEvent event) {
    Data data = parse();

    handler.post(() -> {
        updateUI(data);
    });
}
```

流程：

```text
EventBus(BACKGROUND)
            │
            ▼
     解析数据
            │
            ▼
 Handler.post()
            │
            ▼
 MainLooper
            │
            ▼
 updateUI()
```

因此：

> EventBus负责线程切换和事件分发；  
> Handler负责将最终UI更新切回主线程。

# 四、register() 原理

## 注册

```java
EventBus.getDefault().register(this);
```

源码：

```java
public void register(Object subscriber) {
    Class<?> subscriberClass =
            subscriber.getClass();

    List<SubscriberMethod> subscriberMethods =
            subscriberMethodFinder
                    .findSubscriberMethods(
                            subscriberClass);

    synchronized (this) {
        for (SubscriberMethod method :
                subscriberMethods) {
            subscribe(subscriber,
                    method);
        }
    }
}
```

# enqueue()

源码：

```java
void enqueue(
        Subscription subscription,
        Object event) {

    PendingPost pendingPost =
            PendingPost.obtainPendingPost(
                    subscription,
                    event);

    queue.enqueue(pendingPost);

    sendMessage(
            obtainMessage());
}
```

---

# handleMessage()

源码：

```java
public void handleMessage(
        Message msg) {

    PendingPost pendingPost =
            queue.poll();

    eventBus.invokeSubscriber(
            pendingPost);
}
```

流程：

```text
post()
    │
    ▼
HandlerPoster.enqueue()
    │
    ▼
sendMessage()
    │
    ▼
MainLooper
    │
    ▼
handleMessage()
    │
    ▼
method.invoke()
```

所以面试官最喜欢问：

> EventBus 为什么能更新 UI？

答案：

> MAIN ThreadMode 内部就是 Handler + MainLooper 实现线程切换。

# 十一、为什么 EventBus 注册速度慢

因为：

```java
clazz.getDeclaredMethods()
```

和：

```java
getAnnotation()
```

反射耗时。

---

GreenRobot优化：

```text
Subscriber Index
```

APT生成：

```java
MyEventBusIndex
```

注册：

```java
EventBus.builder()
        .addIndex(
                new MyEventBusIndex()
        )
        .installDefaultEventBus();
```

避免：

```text
反射扫描
```

速度提升数倍。

# 十二、EventBus 会内存泄漏吗

会。

例如：

```java
public class MainActivity
        extends AppCompatActivity {

    @Override
    protected void onCreate() {
        EventBus.getDefault()
                .register(this);
    }
}
```

忘记：

```java
unregister()
```

由于：

```text
subscriptionsByEventType
        ↓
Subscription
        ↓
Activity
```

形成：

```text
EventBus(Singleton)
      │
      ▼
HashMap
      │
      ▼
Activity
```

Activity 无法 GC。

---

正确写法：

```java
@Override
protected void onDestroy() {
    EventBus.getDefault()
            .unregister(this);
}
```

# 面试标准回答

```text
1.EventBus本质是发布订阅模式。

2.register时通过反射扫描@Subscribe，
生成SubscriberMethod，并缓存到
subscriptionsByEventType(HashMap)。

3.post时根据Event类型快速找到Subscription，
再根据ThreadMode进行线程切换。

4.MAIN模式底层是HandlerPoster，
继承Handler并绑定MainLooper，
通过sendMessage切换到UI线程。

5.BACKGROUND和ASYNC底层使用ExecutorService线程池。

6.最终通过Method.invoke反射调用订阅函数。

7.为了避免反射开销，可使用APT生成Subscriber Index。

8.EventBus是进程内通信框架，性能远高于BroadcastReceiver，
但需要注意register/unregister，否则容易造成内存泄漏。
```
