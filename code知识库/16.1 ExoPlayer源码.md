# 二、为什么不用MediaPlayer

MediaPlayer：

```text
Java
  │
JNI
  │
MediaPlayerService
  │
Stagefright
```

缺点：

```text
不可扩展
不支持自定义协议
缓存能力弱
ABR能力有限
源码复杂
```

ExoPlayer：

```text
纯Java实现
支持DASH
支持HLS
支持RTSP
支持缓存
支持DRM
支持ABR
```

面试回答：

> MediaPlayer属于系统播放器，而ExoPlayer是Google实现的可扩展播放器框架，支持自定义数据源、缓存、ABR、自定义解码器，因此成为主流播放器方案。
