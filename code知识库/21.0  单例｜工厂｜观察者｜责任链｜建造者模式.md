
## 1. 单例模式 (Singleton Pattern)

### 核心原理

确保一个类**只有一个实例**，并提供一个全局唯一的访问点。常用于管理全局状态、共享资源或昂贵的对象（如数据库连接、网络客户端）。

### Android 中的应用

- **系统服务**：`InputMethodManager`、`LayoutInflater` 等系统服务均通过单例管理。
    
- **数据库操作**：Room 数据库实例（`AppDatabase`）。
    

### 典型代码（Kotlin）

在 Kotlin 中，`object` 关键字天然实现了线程安全的单例：

Kotlin

```
// 线程安全的双重检查锁（Double-Check Locking）写法
class NetworkManager private constructor(context: Context) {
    companion object {
        @Volatile
        private var INSTANCE: NetworkManager? = null

        fun getInstance(context: Context): NetworkManager {
            return INSTANCE ?: synchronized(this) {
                INSTANCE ?: NetworkManager(context.applicationContext).also { INSTANCE = it }
            }
        }
    }
}
```

## 2. 工厂模式 (Factory Pattern)

### 核心原理

将对象的创建逻辑封装在工厂类中，客户端无需关注具体的实例化细节，只需通过统一接口获取所需对象，实现解耦。

### Android 中的应用

- **`ViewModelProvider.Factory`**：Android Jetpack 中创建带有参数的 `ViewModel`。
    
- **`BitmapFactory`**：解析不同来源（文件、字节流、资源等）并生成 `Bitmap` 对象。
    

### 典型代码

Kotlin

```
// ViewModel 工厂模式应用
class MyViewModelFactory(private val repository: MyRepository) : ViewModelProvider.Factory {
    override fun <T : ViewModel> create(modelClass: Class<T>): T {
        if (modelClass.isAssignableFrom(MyViewModel::class.java)) {
            @Suppress("UNCHECKED_CAST")
            return MyViewModel(repository) as T
        }
        throw IllegalArgumentException("Unknown ViewModel class")
    }
}
```

## 3. 观察者模式 (Observer Pattern)

### 核心原理

定义对象间的一种**一对多**依赖关系，当被观察者（Subject）状态发生改变时，所有依赖于它的观察者（Observer）都会收到通知并自动更新。

### Android 中的应用

- **UI 事件监听**：`Button.setOnClickListener`。
    
- **响应式数据流**：`LiveData`、`StateFlow`、`SharedFlow` 或 `RxJava`。
    

### 典型代码

Kotlin

```
// LiveData 观察者模式
val nameLiveData = MutableLiveData<String>()

// 注册观察者
nameLiveData.observe(viewLifecycleOwner) { newName ->
    textView.text = newName
}

// 被观察者状态改变，自动通知所有观察者
nameLiveData.value = "Gemini"
```

## 4. 责任链模式 (Chain of Responsibility)

### 核心原理

将请求的发送者和接收者解耦，让多个对象都有机会处理该请求。将这些对象连成一条链，并沿着这条链传递请求，直到有对象处理它为止。

### Android 中的应用

- **OkHttp 拦截器 (Interceptor)**：网络请求经过 `RetryAndFollowUpInterceptor` -> `BridgeInterceptor` -> `CacheInterceptor` -> `ConnectInterceptor` -> `CallServerInterceptor` 责任链逐层处理。
    
- **View 事件分发机制**：`dispatchTouchEvent` -> `onInterceptTouchEvent` -> `onTouchEvent` 构成了典型的责任链传递过程。
    

### 典型代码（OkHttp 拦截器）

Kotlin

```
class LoggingInterceptor : Interceptor {
    override fun intercept(chain: Interceptor.Chain): Response {
        val request = chain.request()
        println("Sending request: ${request.url}")
        
        // 将请求传递给链中的下一个拦截器处理
        val response = chain.proceed(request)
        
        println("Received response for: ${response.request.url}")
        return response
    }
}
```

## 5. 建造者模式 (Builder Pattern)

### 核心原理

将一个复杂对象的构建与其表示分离，使得同样的构建过程可以创建不同的表示。主要解决构造函数参数过多、多重重载导致的臃肿问题。

### Android 中的应用

- **`AlertDialog.Builder`**
    
- **`NotificationCompat.Builder`**
    
- **`OkHttpClient.Builder`**、**`Retrofit.Builder`**
    

### 典型代码

Kotlin

```
val dialog = AlertDialog.Builder(context)
    .setTitle("提示")
    .setMessage("确定要删除此项吗？")
    .setPositiveButton("确定") { dialog, _ -> dialog.dismiss() }
    .setNegativeButton("取消", null)
    .create()

dialog.show()
```

## 常见设计模式汇总对比

|**设计模式**|**模式分类**|**核心作用**|**Android 典型应用示例**|
|---|---|---|---|
|**单例模式**|创建型|保证全局仅存在一个对象实例|Room 数据库、`InputMethodManager`|
|**工厂模式**|创建型|封装对象创建逻辑，降低耦合|`ViewModelProvider.Factory`、`BitmapFactory`|
|**建造者模式**|创建型|链式构建参数复杂的复杂对象|`AlertDialog.Builder`、`NotificationCompat.Builder`|
|**观察者模式**|行为型|状态变更时自动通知多个订阅者|`LiveData` / `Flow`、`OnClickListener`|
|**责任链模式**|行为型|将请求沿节点链条传递并依次处理|OkHttp `Interceptor`、触摸事件分发机制|
|**代理模式**|结构型|为其他对象提供一种代理以控制访问|Retrofit（动态代理 `Proxy`）|
