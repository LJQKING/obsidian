**Kotlin 内联函数（Inline Functions）** 是 Kotlin 中一个重要的性能优化特性，主要用于解决高阶函数（尤其是传递 Lambda 表达式）带来的运行时开销。

### 1. 为什么需要内联函数？

普通高阶函数在编译时会将 Lambda 转换为函数对象（FunctionN 接口的实例），这会产生**额外的内存分配和调用开销**。  
内联函数通过 `inline` 关键字让**编译器在调用处直接复制函数体代码**，从而消除这些开销，同时还支持**非局部返回（non-local return）**。

### 2. 基本语法

```kotlin
inline fun <T> myInlineFunction(block: () -> T): T {
    // 函数体
    return block()
}
```

### 3. 经典示例

```kotlin
// 不使用 inline 时
fun lock(lock: Lock, body: () -> Unit) {
    lock.lock()
    try {
        body()          // 这里会创建 Lambda 对象
    } finally {
        lock.unlock()
    }
}

// 使用 inline 后（推荐写法）
inline fun <T> lock(lock: Lock, body: () -> T): T {
    lock.lock()
    try {
        return body()
    } finally {
        lock.unlock()
    }
}

// 调用时
lock(someLock) {
    // 这里的代码会被直接内联到调用处
    println("执行中...")
    return  // 支持非局部返回（直接从调用函数返回）
}
```

### 4. `noinline` 和 `crossinline`

当一个 inline 函数有多个函数类型参数时，你可能不想全部内联：

- **`noinline`**：标记某个 Lambda 参数**不要内联**（仍会生成函数对象）。
- **`crossinline`**：标记某个 Lambda 参数**不允许非局部返回**（常用于回调场景）。

```kotlin
inline fun higherOrder(
    noinline noInlineBlock: () -> Unit,   // 不内联
    crossinline crossBlock: () -> Unit,   // 内联但禁止非局部 return
    inlineBlock: () -> Unit               // 正常内联
) {
    // ...
}
```

### 5. 内联属性（Inline Properties）

```kotlin
inline val <T> List<T>.lastOrNull: T?
    get() = if (isNotEmpty()) this[lastIndex] else null
```

### 6. 使用建议（官方推荐）

- **适合场景**：
  - 高阶函数，特别是频繁调用的工具函数（如 `let`、`apply`、`with` 等标准库函数很多都是 inline）。
  - 需要非局部返回的场景。
  - 代码较短的函数（推荐 1~3 行代码）。

- **不适合场景**：
  - 代码很长的函数（会导致字节码膨胀）。
  - 递归函数（一般不能内联）。
  - public API 中过度使用 inline（会影响二进制兼容性）。

### 7. 性能对比

```kotlin
// 普通版本：每次调用都会创建 Lambda 对象
list.forEach { println(it) }

// 内联版本（标准库中 forEach 是 inline 的）
inline fun <T> Iterable<T>.forEach(action: (T) -> Unit) { ... }
```
