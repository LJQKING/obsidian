**Android MQTT 原理及对应源码分析（基于 Eclipse Paho Android Service）。**

### 1. MQTT 协议原理概述

**MQTT（Message Queuing Telemetry Transport）** 是一种轻量级、基于发布/订阅（Publish/Subscribe）模式的物联网（IoT）消息传输协议。它运行在 TCP/IP 之上（或 WebSocket），设计目标是低带宽、高延迟、资源受限的环境（如传感器、移动设备）。

#### 核心概念：
- **Broker（代理/服务器）**：消息中转站（如 Mosquitto、EMQX、HiveMQ）。客户端不直接通信，而是通过 Broker。
- **Client（客户端）**：发布者（Publisher）或订阅者（Subscriber），或两者兼具。
- **Topic（主题）**：消息分类的字符串层次结构（如 `home/livingroom/temperature`）。支持通配符（`+` 单层、`#` 多层）。
- **Publish/Subscribe**：解耦合。Publisher 发消息到 Topic，Subscriber 接收匹配 Topic 的消息，Broker 负责路由。

#### 关键特性 - QoS（Quality of Service，服务质量）：
MQTT 提供三种消息交付保证级别（Publisher 指定发布 QoS，Subscriber 指定订阅 QoS，最终以较低者为准）：
- **QoS 0（At most once）**：最多交付一次。“即发即忘”，无确认，无重传。适合不重要消息（如实时传感器数据，丢失可接受）。
- **QoS 1（At least once）**：至少交付一次。Publisher 发送 PUBLISH，等待 PUBACK 确认。若无确认则重传。可能重复。
- **QoS 2（Exactly once）**：恰好一次。最可靠（两阶段：PUBREC → PUBREL → PUBCOMP），开销最大，适合计费/命令场景。

其他重要机制：
- **LWT（Last Will and Testament）**：客户端异常断开时，Broker 替它发布遗嘱消息。
- **Keep Alive / 心跳**：客户端定期发送 PINGREQ，Broker 回复 PINGRESP。超时断开。
- **Clean Session**：true 时无状态；false 时 Broker 保留订阅和离线消息。
- **Retained Message**：Broker 保留最后一条消息，新订阅者立即收到。
- **Will / Retained / Duplicate** 等标志位在 PUBLISH 包中携带。

协议包类型（Control Packets）：CONNECT、CONNACK、PUBLISH、PUBACK 等。包头固定 2 字节 + 可变头部 + Payload。

**Android 上的适用性**：Android 设备常处于不稳定网络（WiFi/移动数据切换、休眠），因此需要后台持久连接、自动重连、离线缓存等支持。

### 2. Android 上 MQTT 实现 - Eclipse Paho Android Service

Android 原生没有内置 MQTT，最流行的是 **Eclipse Paho** 项目：
- `org.eclipse.paho.client.mqttv3`：核心 Java MQTT 客户端。
- `org.eclipse.paho.android.service`：Android 专用封装，提供 `Service` 后台运行、Broadcast 通信、Alarm 心跳等。

**集成方式**（Gradle）：
```gradle
repositories {
    maven { url "https://repo.eclipse.org/content/repositories/paho-snapshots/" }
}
dependencies {
    implementation 'org.eclipse.paho:org.eclipse.paho.client.mqttv3:1.2.5' // 或最新
    implementation 'org.eclipse.paho:org.eclipse.paho.android.service:1.1.1'
}
```

添加权限（AndroidManifest.xml）：
```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.WAKE_LOCK" /> <!-- 保持唤醒 -->
<service android:name="org.eclipse.paho.android.service.MqttService" />
```

### 3. 源码分析（核心类）

#### **MqttAndroidClient**（入口类）
- 继承/封装 `MqttClient`，提供 Android 上下文。
- 关键方法：
  - `connect(MqttConnectOptions options)`：异步连接，返回 `IMqttToken`。
  - `subscribe(String topicFilter, int qos)` / `publish(String topic, MqttMessage message)`。
  - 通过 `MqttService` 后台处理。

典型初始化：
```java
String clientId = MqttClient.generateClientId();
MqttAndroidClient client = new MqttAndroidClient(context, "tcp://broker.example.com:1883", clientId);
```

源码中它会启动/绑定 `MqttService`，并通过 BroadcastReceiver 接收回调（`messageArrived`、`connectionLost` 等）。

#### **MqttService**（核心 Service）
- 后台 `Service`，处理实际连接、消息收发。
- `callbackToActivity()`：通过 `sendBroadcast(Intent)` 发送 `CALLBACK_TO_ACTIVITY` 动作，携带消息/状态到 Activity。
- 支持消息持久化（`MessageStore` / `DatabaseMessageStore` 使用 SQLite 缓存离线消息）。
- 管理多个 `MqttConnection`。

#### **MqttConnection**
- 封装实际的 MQTT 逻辑。
- 重写 `MqttCallback`：
  - `messageArrived(String topic, MqttMessage message)`：收到消息后回调 Service。
  - `connectionLost(Throwable cause)`：断线处理（触发重连）。
  - `deliveryComplete(IMqttDeliveryToken token)`：QoS 1/2 确认完成。

#### **AlarmPingSender**（心跳机制）
- Android 特有：使用 `AlarmManager` 定时发送 PINGREQ，适应 Doze 模式和休眠。比纯线程心跳更省电可靠。

#### **其他辅助**：
- `MqttTokenAndroid` / `MqttDeliveryTokenAndroid`：Token 用于异步回调（`IMqttActionListener`）。
- `ParcelableMqttMessage`：支持 Intent 传递消息。
- 自动重连（`Automatic Reconnect`）：在 `MqttConnectOptions` 中配置，结合 Service 生命周期实现。

**连接流程源码简析**：
1. `MqttAndroidClient.connect()` → 启动 `MqttService` 并 bind。
2. Service 创建 `MqttAsyncClient`（底层）执行 CONNECT 包。
3. 成功后 CONNACK，设置回调。
4. 发布/订阅类似，通过底层 Client 发送对应包，QoS 逻辑在 `MqttClient` 中处理（重传、状态机）。

**消息接收流程**：
Broker → TCP → `MqttService` → `messageArrived` → Broadcast Intent → Activity 的 `BroadcastReceiver` → `MqttCallback.messageArrived`。

### 4. 完整使用示例（Kotlin/Java 混合风格）

```kotlin
// 1. 初始化
private lateinit var mqttClient: MqttAndroidClient

fun connect(context: Context) {
    val serverUri = "tcp://broker.hivemq.com:1883"
    val clientId = MqttClient.generateClientId()
    mqttClient = MqttAndroidClient(context.applicationContext, serverUri, clientId)

    val options = MqttConnectOptions().apply {
        isCleanSession = true
        connectionTimeout = 10
        keepAliveInterval = 60
        // userName, password, LWT 等
    }

    mqttClient.connect(options, null, object : IMqttActionListener {
        override fun onSuccess(asyncActionToken: IMqttToken) {
            Log.d("MQTT", "Connected")
            mqttClient.subscribe("test/topic", 1) // QoS 1
        }
        override fun onFailure(asyncActionToken: IMqttToken, exception: Throwable) { ... }
    })

    // 设置回调
    mqttClient.setCallback(object : MqttCallbackExtended {
        override fun messageArrived(topic: String, message: MqttMessage) {
            Log.d("MQTT", "Received: $topic -> ${String(message.payload)}")
        }
        override fun connectionLost(cause: Throwable?) { /* 重连逻辑 */ }
        // ...
    })
}

// 发布
fun publish(topic: String, payload: String, qos: Int = 1) {
    val message = MqttMessage(payload.toByteArray()).apply { this.qos = qos }
    mqttClient.publish(topic, message)
}
```

**注意事项**：
- 在 Service / Application 中管理 Client 生命周期，避免 Activity 销毁导致断连。
- 权限、电池优化（忽略电池优化）、网络变化广播监听。
- 生产环境建议 SSL/TLS（`ssl://`）、认证、离线缓冲。

### 5. 总结与扩展

- **Android 源码亮点**：Service + Alarm 心跳 + Broadcast + 持久化，完美适配移动端。
