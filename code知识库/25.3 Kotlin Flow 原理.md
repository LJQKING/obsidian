# Kotlin Flow 原理（Android 面试源码级解析）

Flow 是 Kotlin 协程提供的**冷流（Cold Stream）**，用于异步数据流处理，可以理解为：

> Flow = 协程版 RxJava Observable + 挂起函数（Suspend）

官方定义：

```kotlin
public interface Flow<out T> {
    public suspend fun collect(
        collector: FlowCollector<T>
    )
}
```

核心特点：

- 支持协程（Suspend）
    
- 支持背压（BackPressure）
    
- 冷流（Cold Stream）
    
- 可取消（Cancellation）
    
- 操作符丰富（map、filter、zip、flatMapLatest）

# 三、Flow源码分析

## flow {}

源码：

```kotlin
public fun <T> flow(
    block: suspend FlowCollector<T>.() -> Unit
): Flow<T> = SafeFlow(block)
```

创建：

```kotlin
SafeFlow
```

---

## SafeFlow

源码：

```kotlin
private class SafeFlow<T>(
    private val block:
        suspend FlowCollector<T>.() -> Unit
) : AbstractFlow<T>() {

    override suspend fun collectSafely(
        collector: FlowCollector<T>
    ) {
        collector.block()
    }
}
```

可以看到：

```kotlin
flow {
    emit(1)
}
```

其实就是：

```kotlin
collector.emit(1)
```

# 四、collect原理

源码：

```kotlin
public abstract class AbstractFlow<T>
    : Flow<T> {

    public final override suspend fun collect(
        collector: FlowCollector<T>
    ) {
        val safeCollector =
            SafeCollector(
                collector,
                coroutineContext
            )

        try {
            collectSafely(safeCollector)
        } finally {
            safeCollector.releaseIntercepted()
        }
    }
}
```

执行：

```
collect()
      │
      ▼
SafeCollector
      │
      ▼
collectSafely()
      │
      ▼
emit()
      │
      ▼
collect{}
```

# 六、为什么说Flow是冷流（Cold Stream）

代码：

```kotlin
val flow = flow {
    println("start")
    emit(1)
}
```

此时：

```
什么都不会执行
```

只有：

```kotlin
flow.collect()
```

才执行：

```
start
1
```

每次 collect：

```kotlin
flow.collect()
flow.collect()
```

输出：

```
start
1

start
1
```

说明：

### 每个collect都会重新执行flow代码块

源码：

```kotlin
override suspend fun collectSafely(
    collector: FlowCollector<T>
) {
    collector.block()
}
```

每次 collect：

```
重新调用
collector.block()
```

所以：

```
Flow
≠ 数据

Flow
= 数据生产逻辑
```

# 七、Flow背压（BackPressure）

生产快：

```kotlin
flow {
    for (i in 0..1000) {
        emit(i)
    }
}
```

消费慢：

```kotlin
collect {
    delay(1000)
}
```

普通回调：

```
OOM
```

Flow：

```kotlin
emit()
```

是：

```kotlin
suspend fun emit()
```

因为 emit 是挂起函数：

```
生产者
      ↓
emit()
      ↓
消费者处理
      ↓
恢复生产
```

天然背压：

```
Producer
    │
emit()
    │ suspend
    ▼
Consumer
```

# 八、flowOn线程切换原理

代码：

```kotlin
flow {
    emit(loadData())
}.flowOn(Dispatchers.IO)
```

collect：

```kotlin
flow.collect {
}
```

执行：

```
emit
      ↓
IO线程

collect
      ↓
Main线程
```

源码：

```kotlin
public fun <T> Flow<T>.flowOn(
    context: CoroutineContext
): Flow<T> {
    return ChannelFlowOperatorImpl(
        this,
        context
    )
}
```

内部：

```
Producer(IO)
       │
    Channel
       │
Consumer(Main)
```

其实：

### flowOn = Channel + CoroutineContext

# 十二、StateFlow原理

```

public interface StateFlow<T>

    val value: T

```

内部：

```

      │

Array<StateFlowSlot>
```

保存所有订阅者：

```

```

更新：

```kotlin

```

遍历：

```
for(slot in slots){
    slot.makePending()
}
```

通知所有 Collector。

# 十五、Android面试高频追问

### 1）Flow为什么支持背压？

因为：

```kotlin
emit()
```

是：

```kotlin
suspend fun emit()
```

生产者会因为消费者慢而挂起。

---

### 2）Flow为什么是冷流？

因为：

```kotlin
flow {}
```

保存的是：

```kotlin
suspend block
```

每次：

```kotlin
collect()
```

重新执行：

```kotlin
collector.block()
```

---

### 3）flowOn为什么能切线程？

本质：

```text
CoroutineContext
       +
Channel
```

上游生产和下游消费运行在不同协程。

---

### 4）StateFlow和LiveData区别

|特性|LiveData|StateFlow|
|---|---|---|
|生命周期感知|✅|❌|
|协程支持|❌|✅|
|冷热流|热流|热流|
|初始值|❌|✅|
|背压|❌|✅|
|操作符|少|丰富|

---

### 5）StateFlow和SharedFlow区别

|特性|StateFlow|SharedFlow|
|---|---|---|
|初始值|必须|不需要|
|value属性|有|无|
|自动去重|有|无|
|replay|固定1|可配置|
|场景|UI状态|事件总线|

# 一句话总结

```text
Flow
= suspend + FlowCollector + SafeCollector

flow{}
= 保存一个 suspend block

collect()
= 执行 block

emit()
= 回调 FlowCollector.emit()

flowOn()
= Channel + CoroutineContext

buffer()
= Channel缓存

StateFlow
= SharedFlow(replay=1) + distinctUntilChanged()

SharedFlow
= 热流 + 多播 + 缓存
```

在 Android 架构中：

```text
UI
 │
collectAsState()
 │
StateFlow
 │
ViewModel
 │
Repository
 │
Flow
 │
Room / Retrofit / DataStore
```

这是目前 Jetpack + MVVM + Compose 官方推荐的响应式数据流方案。
