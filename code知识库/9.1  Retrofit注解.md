Retrofit 通过**注解**把 HTTP 请求映射成 Java/Kotlin 接口方法，非常简洁优雅。

### 1. **请求方法注解**（Method）

| 注解          | 用途                          | 示例 |
|---------------|-------------------------------|------|
| `@GET`        | GET 请求                      | `@GET("users/{id}")` |
| **`@POST`**   | **POST 请求**（最常用）       | `@POST("user/login")` |
| `@PUT`        | PUT 请求（更新）              | `@PUT("users/{id}")` |
| `@DELETE`     | DELETE 请求                   | `@DELETE("users/{id}")` |
| `@PATCH`      | PATCH 请求（部分更新）        | `@PATCH("users/{id}")` |
| `@HEAD`       | HEAD 请求                     | `@HEAD("users/{id}")` |
| `@OPTIONS`    | OPTIONS 请求                  | `@OPTIONS("users")` |

---

### 2. **请求参数注解**（Parameter）

| 注解                  | 用途                              | 示例 |
|-----------------------|-----------------------------------|------|
| **`@Body`**           | **JSON 请求体**（最常用）         | `@Body user: UserRequest` |
| **`@Field`**          | Form 表单字段（需 `@FormUrlEncoded`） | `@Field("username") name: String` |
| **`@FieldMap`**       | Form 表单 Map                    | `@FieldMap map: Map<String, String>` |
| **`@Part`**           | Multipart 单部分（文件/文本）     | `@Part file: MultipartBody.Part` |
| **`@PartMap`**        | Multipart Map                     | - |
| **`@Query`**          | URL 查询参数                      | `@Query("page") page: Int` |
| **`@QueryMap`**       | URL 查询参数 Map                  | `@QueryMap options: Map<String, String>` |
| **`@Path`**           | 路径参数（替换 `{id}`）           | `@Path("id") id: String` |
| `@Header`             | 单次请求 Header                   | `@Header("Authorization") token: String` |
| `@Headers`            | 多个静态 Header                   | `@Headers("Accept: application/json")` |
| `@Url`                | 动态完整 URL（覆盖 baseUrl）      | `@Url url: String` |

---

### 3. **表单 / 上传 专用**

```kotlin
// 1. Form 表单
@FormUrlEncoded
@POST("oauth/token")
suspend fun login(
    @Field("grant_type") grantType: String,
    @Field("username") username: String,
    @Field("password") password: String
): Response<TokenResponse>

// 2. 文件上传
@Multipart
@POST("upload")
suspend fun upload(
    @Part file: MultipartBody.Part,           // 文件
    @Part("description") desc: RequestBody    // 文本
): Response<UploadResult>
```

**创建 MultipartBody.Part 示例**：

```kotlin
val filePart = MultipartBody.Part.createFormData(
    "file", 
    file.name, 
    file.asRequestBody("image/*".toMediaTypeOrNull())
)
```

### 4. **响应处理注解**

- 返回 `Call<T>`：传统方式
- 返回 `suspend fun` + `Response<T>`：**推荐**（协程）
- 返回 `Observable<T>` / `Flowable<T>`：RxJava
- 返回 `LiveData<T>`：需要额外 adapter

### 5. **完整示例**

```kotlin
interface ApiService {

    @POST("user/login")
    suspend fun login(@Body request: LoginRequest): Response<LoginResponse>

    @FormUrlEncoded
    @POST("user/register")
    suspend fun register(
        @FieldMap params: Map<String, String>
    ): Response<BaseResponse>

    @Multipart
    @POST("profile/avatar")
    suspend fun updateAvatar(
        @Part avatar: MultipartBody.Part
    ): Response<UserProfile>

    @GET("users")
    suspend fun getUsers(
        @Query("page") page: Int,
        @Query("size") size: Int = 20
    ): Response<UserListResponse>
}
```

---

### 6. 最佳实践提示

1. **全局 Header / Token** → 用 **OkHttp Interceptor**（推荐），不要每个方法都写 `@Header`。
2. **BaseUrl** 动态切换 → 使用 `@Url` 或多个 Retrofit 实例。
3. **Gson / Moshi** → 配合 `addConverterFactory`。
4. **协程** → 使用 `suspend` + `Response<T>` 处理错误更优雅。
5. **错误统一处理** → 在 OkHttp `Interceptor` 中处理非 2xx 响应。
