## 1. OkHttp（网络底层）

### 核心作用

👉 负责真正的 **HTTP 请求执行层**

### 核心原理

OkHttp 本质是一个 **责任链 + Socket 连接池模型**

请求流程：

```
Request
  ↓
Interceptor Chain（责任链）
  ↓
CacheInterceptor（缓存）
  ↓
ConnectInterceptor（连接复用）
  ↓
CallServerInterceptor（真正发请求）
  ↓
Response
```

### 核心能力

- 连接池复用（减少 TCP 握手）
    
- 支持 HTTP/2
    
- 拦截器机制（日志 / Token / 重试）
    
- 超时控制
    
- 缓存机制
    

---

## 2. Retrofit（网络封装层）

### 核心作用

👉 把 HTTP API **变成 Java 接口调用**

例如：

```java
@GET("user/info")
Call<User> getUser();
```

---

### 核心原理

Retrofit 本质是：

> **动态代理 + 注解解析 + OkHttp封装**

流程：

```
接口方法
  ↓
动态代理 Proxy.newProxyInstance
  ↓
解析 @GET / @POST
  ↓
生成 Request
  ↓
交给 OkHttp 执行
```

---

### 核心组件

- Retrofit（入口）
    
- ServiceMethod（解析注解）
    
- CallAdapter（适配 RxJava / Coroutine）
    
- Converter（Gson / Moshi）
    

---

## 3. RxJava（异步调度层）

### 核心作用

👉 解决 **异步 + 线程切换 + 链式调用**

---

### 核心原理

RxJava =

> **观察者模式 + 操作符链式转换**

结构：

```
Observable（被观察者）
   ↓
Operator（map / flatMap）
   ↓
Scheduler（线程切换）
   ↓
Observer（观察者）
```

---

### 关键能力

- 线程切换（subscribeOn / observeOn）
    
- 链式操作
    
- 数据变换
    
- 合并请求
    

---

# 二、三者组合关系（非常重要）

实际项目中：

```
RxJava 负责调度
Retrofit 负责接口
OkHttp 负责网络
```

完整链路：

```
UI
 ↓
RxJava（线程切换）
 ↓
Retrofit（API）
 ↓
OkHttp（HTTP请求）
 ↓
Server
```


---

## 2️⃣ OkHttp 配置（核心）

```java
OkHttpClient okHttpClient = new OkHttpClient.Builder()
        .connectTimeout(10, TimeUnit.SECONDS)
        .readTimeout(10, TimeUnit.SECONDS)
        .writeTimeout(10, TimeUnit.SECONDS)
        .addInterceptor(chain -> {
            Request original = chain.request();

            Request request = original.newBuilder()
                    .addHeader("token", "123456")
                    .build();

            return chain.proceed(request);
        })
        .build();
```

---

## 3️⃣ Retrofit 创建

```java
Retrofit retrofit = new Retrofit.Builder()
        .baseUrl("https://api.example.com/")
        .client(okHttpClient)
        .addConverterFactory(GsonConverterFactory.create())
        .addCallAdapterFactory(RxJava2CallAdapterFactory.create())
        .build();
```

---

## 4️⃣ 定义 API 接口

```java
public interface ApiService {

    @GET("user/info")
    Observable<User> getUserInfo();

    @POST("user/login")
    @FormUrlEncoded
    Observable<LoginResponse> login(
            @Field("username") String username,
            @Field("password") String password
    );
}
```

---

## 5️⃣ 使用 RxJava 调用网络

```java
ApiService apiService = retrofit.create(ApiService.class);

apiService.getUserInfo()
        .subscribeOn(Schedulers.io())              // IO线程请求
        .observeOn(AndroidSchedulers.mainThread()) // 主线程回调
        .subscribe(new Observer<User>() {
            @Override
            public void onSubscribe(Disposable d) {}

            @Override
            public void onNext(User user) {
                // 更新UI
                Log.d("TAG", user.toString());
            }

            @Override
            public void onError(Throwable e) {
                Log.e("TAG", "error", e);
            }

            @Override
            public void onComplete() {}
        });
```

# 四、进阶组合（面试加分点）

---

## 1️⃣ Retrofit + RxJava 扁平化请求

```java
api.getUser()
   .flatMap(user -> api.getUserDetail(user.id))
   .subscribe(...)
```

👉 用于 **依赖请求**

---

## 2️⃣ 多接口合并（Zip）

```java
Observable.zip(
        api.getUser(),
        api.getOrders(),
        (user, orders) -> new UserWithOrders(user, orders)
)
.subscribe(...)
```

👉 用于首页并行加载

---

## 3️⃣ 统一错误处理（关键）

```java
api.getUser()
   .subscribeOn(Schedulers.io())
   .observeOn(AndroidSchedulers.mainThread())
   .onErrorResumeNext(throwable -> {
       return Observable.just(new User());
   })
   .subscribe(...)
```

---

## 4️⃣ 线程调度封装（工程化）

```java
public class RxSchedulers {

    public static <T> ObservableTransformer<T, T> ioMain() {
        return upstream ->
                upstream.subscribeOn(Schedulers.io())
                        .observeOn(AndroidSchedulers.mainThread());
    }
}
```

使用：

```java
api.getUser()
   .compose(RxSchedulers.ioMain())
   .subscribe(...)
```

# 五、面试一句话总结（非常重要）

可以直接这样答：

> OkHttp 是底层网络请求框架，基于责任链模式实现连接复用与请求拦截；  
> Retrofit 是对 OkHttp 的封装，通过动态代理将接口转为 HTTP 请求；  
> RxJava 是异步响应式框架，负责线程调度和数据流组合；  
> 三者结合后形成：Retrofit 定义接口，OkHttp 执行请求，RxJava 控制异步与线程切换。
