# 第一部分：Gradle构建流程

---

# 1. Gradle 构建流程是什么？

## 标准答案

Gradle本质是一个构建系统。

Android编译过程：

```text
源码
 ↓
AAPT2处理资源
 ↓
生成R.java
 ↓
Java/Kotlin编译
 ↓
.class
 ↓
D8转Dex
 ↓
R8混淆压缩
 ↓
APK打包
 ↓
签名
 ↓
APK
```

---

## 原理分析

Gradle执行三个阶段：

### Initialization

读取：

```text
settings.gradle
```

确定有哪些Module

例如：

```gradle
include ':app'
include ':sdk'
```

---

### Configuration

执行：

```gradle
build.gradle
```

构建Task DAG图

例如：

```gradle
task test {}
```

---

### Execution

执行真正Task：

```text
compile
dex
package
assemble
```

---

## 面试加分

回答：

> Gradle采用DAG有向无环图管理Task，Task之间通过dependsOn形成依赖关系。
