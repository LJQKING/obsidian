工业级蓝牙协议设计，本质不是“能发数据”，而是要解决 4 件事：

- **怎么分包**
    
- **怎么防错**
    
- **怎么确认收到**
    
- **怎么扩展协议**
    

下面给你一套在 BLE / SPP 都通用的“工程级帧结构设计”（很多设备厂都在用这一套思想）。

---

# 一、工业级蓝牙协议核心目标

一个好的协议必须具备：

✔ 可拆包（避免 20 字节限制问题）  
✔ 可校验（防止乱码/干扰）  
✔ 可应答（可靠传输）  
✔ 可扩展（后期加功能不崩）  
✔ 可区分命令（控制/数据/心跳）

---

# 二、标准工业级帧结构（推荐）

## 📌 经典设计（最常见）

```
| HEAD | VER | LEN | CMD | SEQ | DATA | CRC |
```

---

## 📦 字段解释

### 1️⃣ HEAD（帧头，2 bytes）

用于标识数据开始

```
0xAA 0x55
```

👉 用来防粘包 / 找起始位置

---

### 2️⃣ VER（协议版本，1 byte）

```
0x01
```

👉 未来升级协议用

---

### 3️⃣ LEN（数据长度，2 bytes）

```
DATA + CMD + SEQ 的长度
```

👉 用来拆包

---

### 4️⃣ CMD（命令字，1 byte）

核心控制指令：

|CMD|含义|
|---|---|
|0x01|开灯|
|0x02|关灯|
|0x10|读取状态|
|0x20|心跳|
|0x30|设备信息|

---

### 5️⃣ SEQ（序列号，1~2 bytes）

👉 用于：

- 请求/响应匹配
    
- 重发机制
    

```
0x01, 0x02, 0x03...
```

---

### 6️⃣ DATA（可变长度）

真正业务数据：

```
温度 / 控制参数 / JSON / 二进制
```

---

### 7️⃣ CRC（校验，1~2 bytes）

👉 防止数据错误

常用：

- CRC8（轻量 BLE）
    
- CRC16（工业级推荐）
    

---

# 三、完整示例帧

## 📌 示例：开灯命令

```
AA 55 01 00 03 01 01 00 CRC
```

拆解：

|字段|值|
|---|---|
|HEAD|AA 55|
|VER|01|
|LEN|0003|
|CMD|01|
|SEQ|01|
|DATA|00（无参数）|
|CRC|校验值|

# 四、升级版工业协议（推荐）

如果你做 IoT / 工控 / 蓝牙设备，建议用这个：

```
| HEAD | LEN | TYPE | CMD | SEQ | FLAG | DATA | CRC |
```

---

## 📌 FLAG（增强控制）

bit位定义：

```
bit0: 是否需要ACK
bit1: 是否分包
bit2: 是否加密
bit3: 是否压缩
```

---

# 五、BLE 必做：分包机制（重点）

## 📌 因为 BLE 限制 20 bytes（默认）

所以 DATA 必须拆包：

```
[1/3] AA 55 ...
[2/3] AA 55 ...
[3/3] AA 55 ...
```

---

## 📦 分包字段（推荐加）

```
PKG_INDEX  当前包序号
PKG_TOTAL  总包数
```

---

## 示例：

```
AA 55 LEN CMD SEQ 1/3 DATA...
AA 55 LEN CMD SEQ 2/3 DATA...
AA 55 LEN CMD SEQ 3/3 DATA...
```

---

# 六、ACK机制（工业级关键）

## 📌 请求 + 响应结构

### 发送：

```
CMD = 0x01（开灯）
FLAG = 需要ACK
SEQ = 10
```

### 设备返回：

```
CMD = 0x81（0x01 + 0x80 表示响应）
SEQ = 10
RESULT = 0x00 成功
```

---

## 📌 状态码：

|code|含义|
|---|---|
|0x00|成功|
|0x01|参数错误|
|0x02|设备忙|
|0x03|CRC错误|

---

# 七、心跳机制（必须有）

```
CMD = 0x20
DATA = timestamp
```

作用：

- 判断设备是否在线
    
- 自动重连
    
- 保活连接
    

---

# 八、加密（工业项目常见）

简单版：

```
DATA = AES128(原始数据)
```

进阶：

- CRC + AES
    
- 或 Xor + CRC（轻量设备）
    

---

# 十、工业级协议设计总结

一个成熟蓝牙协议 =

✔ 帧头（防粘包）

✔ CMD（功能）  
✔ SEQ（追踪）

✔ 分包（BLE必备）
