# Android Activity 启动模式详解（特性 + 原理 + 面试重点）

Android 提供 4 种启动模式：

```xml
<activity
    android:name=".MainActivity"
    android:launchMode="standard"/>
```

可选值：

```java
standard
singleTop
singleTask
singleInstance
```

---

# 一、为什么需要启动模式

默认情况下：

```java
startActivity(new Intent(this, AActivity.class));
```

每调用一次：

```java
new ActivityRecord()
new Activity()
```

都会创建新的 Activity 实例压入任务栈（Task）。

例如：

```text
A
↓
A
↓
A
```

会造成：

- 重复页面
    
- 内存浪费
    
- 返回栈混乱
    

因此 Android 提供 LaunchMode 控制 Activity 的创建行为。

---

# 二、Task（任务栈）概念

先理解任务栈：

```text
Task1

Top
↓
C
↓
B
↓
A
Bottom
```

特点：

### 后进先出（LIFO）

```java
finish();
```

移除栈顶：

```text
B
↓
A
```

Back键：

```java
finish()
```

本质也是弹栈。

---

# 三、standard（默认模式）

## 特点

每次启动都创建新实例。

```xml
android:launchMode="standard"
```

---

## 示例

当前栈：

```text
A
```

执行：

```java
startActivity(A)
```

结果：

```text
A
↓
A
```

再次启动：

```text
A
↓
A
↓
A
```

---

## 生命周期

旧Activity：

```java
onPause()
```

新Activity：

```java
onCreate()
onStart()
onResume()
```

---

## 适用场景

大多数业务页面：

- 商品详情
    
- 新闻详情
    
- 图片浏览
    

例如：

```text
商品1
↓
商品2
↓
商品3
```

允许重复存在。

---

# 四、singleTop

## 特点

如果目标Activity已经位于栈顶：

```text
Top
↓
A
```

再次启动：

```java
startActivity(A)
```

不会创建新实例。

而是：

```java
onNewIntent()
```

---

## 示例1（栈顶）

当前：

```text
A
```

启动：

```java
startActivity(A)
```

结果：

```text
A
```

不会创建。

调用：

```java
onNewIntent()
```

---

## 示例2（不在栈顶）

当前：

```text
B
↓
A
```

启动：

```java
startActivity(A)
```

结果：

```text
B
↓
A
↓
A
```

仍然创建新实例。

---

## 生命周期

不创建时：

```java
onNewIntent()
onResume()
```

---

## 典型场景

通知栏点击：

```java
PendingIntent
```

避免：

```text
Main
↓
Main
↓
Main
```

---

# 五、singleTask

面试最爱问。

---

## 特点

任务栈中只允许存在一个实例。

如果存在：

```text
A
↓
B
↓
C
```

启动：

```java
startActivity(B)
```

系统会找到 B。

销毁 B 上面的所有 Activity：

```text
A
↓
B
```

然后：

```java
onNewIntent()
```

---

## 示例

当前：

```text
A
↓
B
↓
C
↓
D
```

启动：

```java
startActivity(B)
```

结果：

```text
A
↓
B
```

C 和 D 被移除。

---

## 生命周期

```java
D.onDestroy()
C.onDestroy()

B.onNewIntent()
B.onRestart()
B.onStart()
B.onResume()
```

---

## 典型场景

首页：

```java
MainActivity
```

登录页：

```java
LoginActivity
```

消息中心：

```java
MessageActivity
```

不希望重复创建。

---

# 六、singleInstance

最特殊。

---

## 特点

独占一个任务栈。

系统单独开 Task。

---

## 示例

启动：

```java
VideoActivity
```

系统：

```text
Task1
A
↓
B

Task2
VideoActivity
```

---

再次启动：

```java
startActivity(VideoActivity)
```

直接复用。

---

## 启动其他页面

如果：

```java
VideoActivity
```

启动：

```java
DetailActivity
```

则：

```text
Task1
A
↓
B
↓
DetailActivity

Task2
VideoActivity
```

Detail 不会进入 Video 所在栈。

---

## 场景

历史：

- 来电界面
    
- 系统拨号界面
    
- 桌面 Launcher
    

Android 12 后已很少使用。

官方更推荐：

```java
singleTask
```

# 八、启动模式核心源码

Android Framework：

```java
ActivityStarter.java
```

关键逻辑：

```java
switch (launchMode) {

case LAUNCH_SINGLE_TOP:
    // 栈顶复用
    break;

case LAUNCH_SINGLE_TASK:
    // 栈内查找
    break;

case LAUNCH_SINGLE_INSTANCE:
    // 独立Task
    break;

default:
    // standard
}
```

# 九、FLAG 与启动模式关系

很多面试官会追问。

---

## FLAG_ACTIVITY_SINGLE_TOP

效果：

```java
singleTop
```

等价：

```java
intent.addFlags(
FLAG_ACTIVITY_SINGLE_TOP
);
```

---

## FLAG_ACTIVITY_CLEAR_TOP

效果：

```java
singleTask
```

类似：

```java
A
↓
B
↓
C
↓
D
```

启动：

```java
B
```

结果：

```text
A
↓
B
```

---

## FLAG_ACTIVITY_NEW_TASK

新建Task：

```java
intent.addFlags(
Intent.FLAG_ACTIVITY_NEW_TASK
);
```

常用于：

```java
Service
BroadcastReceiver
```

启动Activity。

# 十、面试高频总结

|模式|是否创建新实例|是否调用onNewIntent|是否清除上层Activity|是否独立Task|
|---|---|---|---|---|
|standard|每次创建|否|否|否|
|singleTop|栈顶复用|是|否|否|
|singleTask|栈内唯一|是|是|否（可结合TaskAffinity）|
|singleInstance|全局唯一|是|是|是|

### 一句话记忆

- **standard**：永远创建
    
- **singleTop**：栈顶复用
    
- **singleTask**：栈内唯一，清栈复用
    
- **singleInstance**：独占任务栈
