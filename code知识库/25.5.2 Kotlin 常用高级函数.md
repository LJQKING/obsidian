**Kotlin 常用高级函数（Higher-Order Functions）详解**

Kotlin 的高级函数主要指**接受函数作为参数或返回函数的函数**，最常用的是**集合扩展函数**和**作用域函数（Scope Functions）**。这些函数极大提升了代码的简洁性和函数式编程风格。

### 1. 集合操作类高级函数（最常用）

这些函数是 `Iterable`、`Collection`、`List`、`Map` 等类型的扩展函数，内部大多使用 `inline` 实现，避免函数调用开销。

#### **map / mapNotNull / mapIndexed**

```kotlin
list.map { it * 2 }
list.mapIndexed { index, item -> "$index: $item" }
list.mapNotNull { it.toIntOrNull() }  // 过滤 null
```

**原理**：遍历集合，对每个元素应用转换函数，返回**新的 List**（不修改原集合）。

**适用场景**：
- 数据转换（DTO → VO、实体类映射）
- 格式化输出
- 从对象列表中提取某个属性

#### **filter / filterNot / filterIsInstance / filterNotNull**

```kotlin
list.filter { it > 10 }
list.filterIsInstance<String>()  // 类型过滤
```

**原理**：遍历并保留满足 predicate（条件函数）的元素。

**适用场景**：
- 条件筛选（搜索、权限过滤、数据清洗）
- 去除无效数据

#### **flatMap**

```kotlin
listOf(listOf(1,2), listOf(3,4)).flatMap { it }
```

**原理**：先 `map` 再 `flatten`，常用于嵌套集合展开。

**适用场景**：
- 处理嵌套列表（如 JSON 解析后的数据）
- 数据库多表关联后扁平化结果

#### **fold / reduce**

```kotlin
val sum = list.fold(0) { acc, i -> acc + i }     // 带初始值
val sum2 = list.reduce { acc, i -> acc + i }    // 无初始值（第一个元素作为初始）
```

**原理**：累积计算，从左到右（也有 `foldRight`）。

**适用场景**：
- 求和、求积、字符串拼接
- 复杂聚合（如计算总价、统计分组结果）

#### **groupBy / groupingBy**

```kotlin
users.groupBy { it.age / 10 * 10 }  // 按年龄段分组
```

**原理**：根据 key 函数分组，返回 `Map<K, List<V>>`。

**适用场景**：
- 数据统计、报表生成、分组展示

#### **sortedBy / sortedWith**

```kotlin
list.sortedBy { it.age }
list.sortedWith(compareBy({it.age}, {it.name}))
```

**适用场景**：排序列表（支持多字段排序）

#### **any / all / none / count / find / firstOrNull**

```kotlin
list.any { it > 0 }     // 存在任意一个
list.all { it > 0 }     // 全部满足
list.find { it.id == targetId }
```

**适用场景**：判断、搜索、校验

### 2. 作用域函数（Scope Functions）—— 最具 Kotlin 特色的

| 函数   | 返回值          | 上下文对象 | 常用场景                     | 推荐使用场景 |
|--------|-----------------|------------|------------------------------|--------------|
| **let**   | Lambda 结果     | `it`       | null 安全调用、链式操作     | **最常用** |
| **run**   | Lambda 结果     | `this`     | 代码块执行、计算结果         | 需要返回值时 |
| **with**  | Lambda 结果     | `this`     | 对同一对象多次操作           | 非空对象多次调用 |
| **apply** | 对象本身        | `this`     | 对象初始化、配置             | **构建对象** |
| **also**  | 对象本身        | `it`       | 附加副作用（如日志）         | 链式中加日志 |

#### 示例

```kotlin
// let + 空安全
val length = user?.let { it.name.length } ?: 0

// apply 初始化
val person = Person().apply {
    name = "张三"
    age = 18
    address = "北京"
}

// also 副作用
list.also { println("处理前: $it") }
    .filter { ... }
    .also { println("处理后: $it") }
```

**原理**：这些函数内部都是 `inline` 的 lambda 扩展，`this` 或 `it` 指向调用者。

### 3. 其他常用高级函数

- **`takeIf / takeUnless`**：条件成立返回对象，否则 null
  ```kotlin
  val result = value.takeIf { it > 0 }
  ```

- **`runCatching`**：优雅处理异常
  ```kotlin
  val result = runCatching { riskyOperation() }
      .getOrDefault(defaultValue)
  ```

- **`use`**（针对 Closeable）
  ```kotlin
  File("test.txt").inputStream().use { ... }
  ```

- **`repeat`、`generateSequence`**

---

### 4. 高级函数的底层原理要点

1. **inline**：大多数集合函数和作用域函数都标记为 `inline`，编译时会把 lambda 代码复制到调用处，避免函数调用和对象分配开销。
2. **Lambda 表达式**：本质是 `FunctionN` 接口的实例，inline 后会被展开。
3. **惰性求值**：`Sequence`（如 `asSequence()`）配合这些函数可实现链式惰性计算，适合大数据量。
4. **不变性**：大部分操作返回**新集合**，符合函数式编程思想。

---

### 5. 实际开发推荐用法场景

- **数据处理层**：`map + filter + groupBy + fold`
- **领域模型构建**：`apply` + `also`
- **空安全链式调用**：`?.let { }?.run { }`
- **集合转换管道**：`asSequence().filter{}.map{}.toList()`
- **日志/调试**：`also`、`onEach`
- **配置对象**（Builder 替代）：`apply`
- **复杂业务计算**：`fold`、`runningFold`（Kotlin 1.4+）
