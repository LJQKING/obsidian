# Fragment 生命周期（完整详解）

Fragment 的生命周期比 Activity 更复杂，因为：

- Fragment 依附于 Activity
    
- Fragment 有自己的生命周期
    
- Fragment 还有 View 生命周期
    
- Fragment 可以被添加、隐藏、替换、回退栈管理
    

---

# 一、Fragment 生命周期总览

## 创建阶段

```java
onAttach()
↓
onCreate()
↓
onCreateView()
↓
onViewCreated()
↓
onViewStateRestored()
↓
onStart()
↓
onResume()
```

## 销毁阶段

```java
onPause()
↓
onStop()
↓
onSaveInstanceState()
↓
onDestroyView()
↓
onDestroy()
↓
onDetach()
```

# 三、每个生命周期详解

---

## 1. onAttach()

Fragment 与 Activity 建立关联。

```java
@Override
public void onAttach(Context context) {
    super.onAttach(context);
}
```

执行时机：

```text
Activity
    ↓
FragmentManager
    ↓
onAttach
```

用途：

- 获取 Activity
    
- 初始化接口回调
    

例如：

```java
mActivity = (MainActivity) context;
```

---

## 2. onCreate()

Fragment 创建。

```java
@Override
public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
}
```

用途：

- 初始化数据
    
- ViewModel 初始化
    
- Adapter 初始化
    

不要：

```java
findViewById()
```

因为 View 还没创建。

---

## 3. onCreateView()

创建 Fragment UI。

```java
@Override
public View onCreateView(
        LayoutInflater inflater,
        ViewGroup container,
        Bundle savedInstanceState) {

    return inflater.inflate(
            R.layout.fragment_home,
            container,
            false);
}
```

作用：

```text
XML
 ↓
inflate
 ↓
View
```

---

## 4. onViewCreated()

View 创建完成。

```java
@Override
public void onViewCreated(
        @NonNull View view,
        Bundle savedInstanceState) {

    super.onViewCreated(view, savedInstanceState);
}
```

这里最常写代码：

```java
RecyclerView rv =
        view.findViewById(R.id.rv);

Button btn =
        view.findViewById(R.id.btn);
```

绑定：

```java
ViewBinding
DataBinding
```

---

## 5. onViewStateRestored()

恢复 View 状态。

```java
@Override
public void onViewStateRestored(
        Bundle savedInstanceState) {
    super.onViewStateRestored(savedInstanceState);
}
```

例如：

```java
EditText内容恢复
RecyclerView位置恢复
```

---

## 6. onStart()

Fragment 可见。

```java
@Override
public void onStart() {
    super.onStart();
}
```

特点：

```text
用户看得到
但不能交互
```

---

## 7. onResume()

前台运行状态。

```java
@Override
public void onResume() {
    super.onResume();
}
```

此时：

```text
可见
可点击
可交互
```

适合：

```java
开始定位
开始相机
开始蓝牙扫描
开始Socket连接
```

---

# 四、退出生命周期

---

## 8. onPause()

失去焦点。

```java
@Override
public void onPause() {
    super.onPause();
}
```

例如：

```text
弹Dialog
启动新Activity
```

处理：

```java
暂停动画
暂停视频
```

---

## 9. onStop()

完全不可见。

```java
@Override
public void onStop() {
    super.onStop();
}
```

适合：

```java
停止定位
停止蓝牙扫描
停止传感器
```

---

## 10. onSaveInstanceState()

保存状态。

```java
@Override
public void onSaveInstanceState(
        @NonNull Bundle outState) {

    super.onSaveInstanceState(outState);
}
```

保存：

```java
outState.putString("name", name);
```

恢复：

```java
savedInstanceState.getString("name");
```

---

## 11. onDestroyView()

销毁 View。

```java
@Override
public void onDestroyView() {
    super.onDestroyView();
}
```

重点：

```text
View没了
Fragment还在
```

这是面试高频。

例如：

```java
replace()
加入回退栈
```

会调用：

```text
onDestroyView
```

不会调用：

```text
onDestroy
```

---

### ViewBinding 防泄漏

```java
private FragmentHomeBinding binding;

@Override
public void onDestroyView() {
    super.onDestroyView();

    binding = null;
}
```

否则容易：

```text
Fragment
 ↓
binding
 ↓
View
 ↓
Activity

形成引用链
```

造成内存泄漏。

---

## 12. onDestroy()

Fragment 对象销毁。

```java
@Override
public void onDestroy() {
    super.onDestroy();
}
```

释放：

```java
Presenter
ViewModel外资源
线程
Socket
```

---

## 13. onDetach()

与 Activity 分离。

```java
@Override
public void onDetach() {
    super.onDetach();
}
```

此时：

```java
getActivity()
```

返回：

```java
null
```

---

# 五、Fragment 与 View 生命周期区别（面试重点）

很多人答错。

## Fragment 生命周期

```text
onAttach
↓
onCreate
↓
onDestroy
↓
onDetach
```

## View 生命周期

```text
onCreateView
↓
onViewCreated
↓
onDestroyView
```

---

例如：

```java
replace(Fragment)
.addToBackStack()
```

执行：

```text
旧Fragment

onPause
onStop
onDestroyView
```

但：

```text
onDestroy
不执行
```

因为 Fragment 实例仍保留在回退栈中。

---

# 六、ViewPager 生命周期

ViewPager2 默认：

```java
FragmentStateAdapter
```

生命周期：

```text
当前页
    RESUMED

左右页
    STARTED
```

Google 官方优化：

```java
setMaxLifecycle()
```

控制：

```text
RESUMED
STARTED
CREATED
```

# 七、源码角度

核心类：

```java
FragmentManager
```

负责：

```text
Fragment事务
生命周期分发
回退栈
状态保存
```

---

执行：

```java
commit()
```

最终：

```java
FragmentManager
    ↓
FragmentStateManager
    ↓
moveToExpectedState()
```

源码：

```java
FragmentStateManager.java
```

生命周期推进：

```java
ATTACHED
CREATED
VIEW_CREATED
STARTED
RESUMED
```

源码中：

```java
moveToExpectedState()
```

会不断比较：

```java
当前状态
目标状态
```

然后执行：

```java
performAttach()
performCreate()
performCreateView()
performStart()
performResume()
```

# Android 面试必问 5 题

### 1. onDestroyView 和 onDestroy 区别？

```text
onDestroyView：
销毁View

onDestroy：
销毁Fragment对象
```

---

### 2. 为什么 ViewBinding 要在 onDestroyView 置空？

```java
binding = null;
```

避免：

```text
Fragment持有View引用
导致内存泄漏
```

---

### 3. Fragment replace 为什么不走 onDestroy？

```java
addToBackStack()
```

时：

```text
仅销毁View

Fragment实例保留
```

---

### 4. Fragment 生命周期和 Activity 生命周期关系？

```text
Activity.onCreate
    ↓
Fragment.onAttach
    ↓
Fragment.onCreate
```

Activity 销毁时：

```text
Fragment全部销毁
```

---

### 5. FragmentManager 如何驱动生命周期？

核心源码链路：

```text
FragmentTransaction.commit()

    ↓

FragmentManager

    ↓

FragmentStateManager

    ↓

moveToExpectedState()

    ↓

performResume()
performPause()
performDestroyView()
...
```

这是 Android Framework 面试中关于 Fragment 生命周期最完整、最常考的知识体系，包括生命周期顺序、View 生命周期、回退栈机制、ViewPager2 生命周期以及 `FragmentManager → FragmentStateManager → moveToExpectedState()` 的源码执行流程。
