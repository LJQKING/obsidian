Android 事件分发和事件拦截，本质是 **Touch Event 在 View 树中的传递规则**，核心围绕三个方法：

- `dispatchTouchEvent()`
    
- `onInterceptTouchEvent()`
    
- `onTouchEvent()`
    

下面按“从上到下 + 再回流”讲清楚。

---

# 一、整体流程（谁先接到事件？）

一次触摸事件（如 ACTION_DOWN）会从：

**Activity → Window → DecorView → ViewGroup → View**

逐层向下传递。

核心链路：

```
Activity.dispatchTouchEvent()
        ↓
PhoneWindow
        ↓
DecorView
        ↓
ViewGroup.dispatchTouchEvent()
        ↓
View.dispatchTouchEvent()
```

---

# 二、三个核心方法作用

## 1️⃣ dispatchTouchEvent（分发事件）

👉 作用：**决定事件往哪里走**

- ViewGroup：负责“要不要拦截 + 分发给子View”
    
- View：负责“处理 or 不处理”
    

返回值：

- true：事件被消费
    
- false：继续向上传/或结束
    

---

## 2️⃣ onInterceptTouchEvent（拦截事件）🔥重点

👉 **只有 ViewGroup 才有这个方法**

作用：决定是否“截胡”事件，不让子View收到

返回值：

|返回值|含义|
|---|---|
|true|拦截事件（自己处理）|
|false|不拦截（继续给子View）|

---

## 3️⃣ onTouchEvent（处理事件）

👉 真正“消费事件”的地方

返回值：

|返回值|含义|
|---|---|
|true|事件被消费（事件链结束）|
|false|不处理，事件往上传|

---

# 三、完整事件分发流程（重点理解）

以 ACTION_DOWN 为例：

```
ViewGroup.dispatchTouchEvent()
    ↓
onInterceptTouchEvent() ?
    ↓
    ├── true（拦截）
    │       ↓
    │   ViewGroup.onTouchEvent()
    │
    └── false（不拦截）
            ↓
        子View.dispatchTouchEvent()
                ↓
            子View.onTouchEvent()
```

---

# 四、关键机制（很多人卡在这里）

## ⭐ 1. DOWN事件决定“事件归属”

如果子View消费了 DOWN：

👉 后续 MOVE / UP 都只给它

如果没消费：

👉 后续事件不会再给它

---

## ⭐ 2. 一旦拦截，子View直接失去后续事件

```java
onInterceptTouchEvent() -> true
```

效果：

- 子View收不到 MOVE / UP
    
- 当前事件序列交给父View处理
    

---

## ⭐ 3. 子View可以“请求父不拦截”

```java
requestDisallowInterceptTouchEvent(true)
```

常用于：

- RecyclerView 内嵌 ScrollView
    
- 横向滑动冲突处理
    

---

# 五、经典事件冲突场景

## 📌 1. ScrollView vs RecyclerView

问题：

- 谁滚？
    

解决：

- 通过 `onInterceptTouchEvent` 判断滑动方向
    

---

## 📌 2. ViewPager 横向 vs ScrollView 纵向

逻辑：

```
横向滑动 → ViewPager拦截
纵向滑动 → ScrollView拦截
```

---

# 六、源码级关键点（面试常问）

## ViewGroup dispatchTouchEvent 核心逻辑：

伪代码：

```java
if (action == DOWN) {
    onInterceptTouchEvent();
}

if (intercepted) {
    return onTouchEvent();
} else {
    child.dispatchTouchEvent();
}
```

---

## 事件传递特点总结：

✔ 单向传递（父 → 子）  
✔ DOWN决定归属  
✔ MOVE/UP跟随DOWN  
✔ 可以被拦截（onInterceptTouchEvent）  
✔ 可以被消费（onTouchEvent）

---

# 七、一句话总结（面试神器）

👉 Android 事件分发机制：

> 事件先从 Activity 向下分发，ViewGroup 可以通过 onInterceptTouchEvent 拦截事件，一旦子 View 消费了 ACTION_DOWN，后续事件都会交给该 View 处理，否则由父容器处理。
