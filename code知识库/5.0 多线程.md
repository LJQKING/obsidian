# Android 多线程详解

Android 中的多线程主要用于**避免主线程（UI线程）阻塞**，提高应用流畅度。

---

## 为什么需要多线程？

Android 规定：

- UI更新必须在主线程执行
    
- 主线程执行耗时操作会导致界面卡顿
    
- 主线程阻塞超过 5 秒可能触发 ANR（Application Not Responding）
    

例如：

```java
// 错误示例
button.setOnClickListener(v -> {
    // 网络请求
    requestServer();

    // 数据库查询
    queryDatabase();

    // 文件下载
    downloadFile();
});
```

如果这些操作耗时较长：

```text
主线程
│
├── UI绘制
├── 点击事件
├── 网络请求（阻塞）
├── 数据库查询（阻塞）
└── 文件下载（阻塞）

=> 页面卡死
=> ANR
```

---

# Android线程模型

```text
┌────────────────────┐
│ Main Thread(UI线程) │
└─────────┬──────────┘
          │
          │ Handler
          ▼
┌────────────────────┐
│ Worker Thread      │
└────────────────────┘
```

主要包括：

|线程|作用|
|---|---|
|Main Thread|UI操作|
|Worker Thread|耗时任务|
|Binder Thread|跨进程通信|
|Render Thread|界面绘制|
|GC Thread|垃圾回收|

# 1. Thread

Java最基础线程

## 创建方式

```java
new Thread(new Runnable() {
    @Override
    public void run() {

        Log.d("TAG","执行任务");

    }
}).start();
```

Java8：

```java
new Thread(() -> {

    Log.d("TAG","执行任务");

}).start();
```

---

## 原理

```text
start()

 ↓

nativeCreate()

 ↓

Linux pthread

 ↓

CPU调度
```

每创建一个 Thread：

- 分配线程栈
    
- 创建内核线程
    
- CPU调度
    

开销较大。

---

# 2. Handler + Looper

Android最经典线程通信方案。

---

## 主线程消息机制

```text
Main Thread

Looper
   │
MessageQueue
   │
Handler
```

系统启动：

```java
ActivityThread.main()
```

执行：

```java
Looper.prepareMainLooper();

Looper.loop();
```

进入无限循环：

```java
for(;;){

    Message msg = queue.next();

    msg.target.dispatchMessage(msg);

}
```

---

## 子线程更新UI

```java
Handler handler = new Handler(Looper.getMainLooper());

new Thread(() -> {

    String result = "成功";

    handler.post(() -> {

        textView.setText(result);

    });

}).start();
```

---

# 3. HandlerThread

Handler + Thread 封装。

适合：

- 串行任务
    
- 消息队列任务
    

---

## 使用

```java
HandlerThread handlerThread =
        new HandlerThread("work");

handlerThread.start();

Handler handler =
        new Handler(handlerThread.getLooper());

handler.post(() -> {

    Log.d("TAG","后台执行");

});
```

---

## 原理

```text
HandlerThread.start()

     ↓

Thread.run()

     ↓

Looper.prepare()

     ↓

Looper.loop()
```

自动创建：

- Looper
    
- MessageQueue

# 4. AsyncTask（已废弃）

Android 11 已废弃。

曾经使用：

```java
class MyTask extends AsyncTask<Void,Void,String>{

    @Override
    protected String doInBackground(Void... params){

        return request();

    }

    @Override
    protected void onPostExecute(String result){

        textView.setText(result);

    }
}
```

废弃原因：

- 生命周期问题
    
- 内存泄漏
    
- 容易崩溃

# 5. Executor线程池

实际开发最常用。

---

## 单线程池

```java
ExecutorService executor =
        Executors.newSingleThreadExecutor();
```

任务串行执行：

```text
任务1
 ↓
任务2
 ↓
任务3
```

---

## 固定线程池

```java
ExecutorService executor =
        Executors.newFixedThreadPool(4);
```

适合：

- 网络请求
    
- 图片加载
    

---

## 缓存线程池

```java
ExecutorService executor =
        Executors.newCachedThreadPool();
```

特点：

```text
线程可复用

需要时创建

空闲60秒销毁
```

---

## 自定义线程池（面试重点）

```java
ThreadPoolExecutor executor =
        new ThreadPoolExecutor(
                4,
                8,
                60,
                TimeUnit.SECONDS,
                new LinkedBlockingQueue<>()
        );
```

参数：

```java
corePoolSize
```

核心线程数

```java
maximumPoolSize
```

最大线程数

```java
keepAliveTime
```

空闲存活时间

```java
workQueue
```

任务队列

---

### 执行流程

```text
提交任务

     │

线程数 < corePoolSize
     │
     ▼
创建核心线程

     │

否则

     ▼
进入队列

     │

队列满

     ▼

创建非核心线程

     │

达到最大线程

     ▼

拒绝策略
```

# 6. Future

```java
        });

String result = future.get();
```

结果：

```text
成功
```

# Android 面试高频问题

### 1. Thread 和 HandlerThread 区别？

Thread：

```text
没有Looper
无法处理消息
```

HandlerThread：

```text
自带Looper
自带MessageQueue
支持Handler通信
```

---

### 2. 为什么Handler会导致内存泄漏？

非静态内部类 Handler：

```java
Handler handler = new Handler(){
};
```

隐式持有：

```text
Handler
   ↓
Activity
```

消息未处理完：

```text
MessageQueue
   ↓
Handler
   ↓
Activity
```

导致 Activity 无法回收。

解决：

```java
static class MyHandler
        extends Handler {

    WeakReference<Activity> ref;
}
```

---

### 3. ThreadPoolExecutor 为什么比 Executors 好？

Executors 默认队列：

```java
LinkedBlockingQueue(Integer.MAX_VALUE)
```

可能导致：

```text
OOM
```

推荐：

```java
new ThreadPoolExecutor(...)
```

自行控制：

- 核心线程数
    
- 最大线程数
    
- 队列大小
    

---

### 4. ANR 本质是什么？

ANR 本质：

```text
主线程长时间无法处理消息
```

常见超时：

|类型|时间|
|---|---|
|Input事件|5s|
|BroadcastReceiver|10s~60s|
|Service|20s|
