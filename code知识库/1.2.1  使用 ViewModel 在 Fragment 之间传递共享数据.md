**使用 ViewModel 在 Fragment 之间传递/共享数据** 是 Android 官方推荐的最佳实践之一。它比 `interface` 回调、 `Bundle`、EventBus 等方式更优雅、生命周期安全。

### 一、核心原理

1. **ViewModel 的作用域（Scope）**：
   - ViewModel 不是直接绑定在 Fragment 上的，而是绑定在 **ViewModelStoreOwner** 上（Activity、Fragment、Navigation Graph 等都可以是 Owner）。
   - 同一个 Owner 返回的 **同一个 ViewModel 类** 会是**同一个实例**。

2. **Fragment 间共享的本质**：
   - 让两个 Fragment 通过**共同的父级 Owner**（通常是宿主 Activity）来获取同一个 ViewModel 实例。
   - Fragment A 修改 ViewModel 中的数据 → Fragment B 立即通过 `LiveData` / `StateFlow` 观察到变化。

3. **为什么推荐这种方式**：
   - 自动处理配置变更（旋转屏幕等）不丢失数据。
   - 解耦：两个 Fragment 不需要直接持有对方引用。
   - 生命周期安全：ViewModel 会在 Activity/Fragment 真正销毁时才被清除。

---

### 二、实现方式（Kotlin 推荐写法）

#### 1. 创建共享的 ViewModel

```kotlin
// SharedViewModel.kt
class SharedViewModel : ViewModel() {

    // 方法一：LiveData（兼容性最好）
    val message: MutableLiveData<String> = MutableLiveData()
    
    // 方法二：StateFlow（更现代，推荐）
    private val _messageFlow = MutableStateFlow("")
    val messageFlow: StateFlow<String> = _messageFlow.asStateFlow()

    fun sendMessage(text: String) {
        message.value = text
        _messageFlow.value = text
    }
}
```

#### 2. 两个 Fragment 获取同一个 ViewModel

**Fragment A（发送方）**：

```kotlin
class FragmentA : Fragment() {

    // 方式一：使用 activityViewModels() 委托（最简洁）
    private val sharedViewModel: SharedViewModel by activityViewModels()

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        button.setOnClickListener {
            sharedViewModel.sendMessage("Hello from Fragment A!")
        }
    }
}
```

**Fragment B（接收方）**：

```kotlin
class FragmentB : Fragment() {

    private val sharedViewModel: SharedViewModel by activityViewModels()

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        // 观察 LiveData
        sharedViewModel.message.observe(viewLifecycleOwner) { msg ->
            textView.text = msg
        }

        // 或者观察 StateFlow（推荐）
        lifecycleScope.launch {
            repeatOnLifecycle(Lifecycle.State.STARTED) {
                sharedViewModel.messageFlow.collect { msg ->
                    textView.text = msg
                }
            }
        }
    }
}
```

### 三、其他获取 ViewModel 的方式

| 方式                        | 使用场景                              | 代码示例 |
|---------------------------|-------------------------------------|---------|
| `activityViewModels()`    | 最常用，共享到 Activity 级别           | `by activityViewModels()` |
| `viewModels()`            | 仅 Fragment 自己使用                   | `by viewModels()` |
| `ViewModelProvider(requireActivity())` | 传统写法                        | `ViewModelProvider(requireActivity())[SharedViewModel::class.java]` |
| Navigation Graph Scope    | 在同一个 NavGraph 中的多个 Fragment 共享 | `by navGraphViewModels(R.id.nav_graph)` |

---

### 四、完整示例结构

```
MainActivity
   ├── FragmentA
   └── FragmentB
```

两个 Fragment 通过 `MainActivity` 作为共同的 ViewModelStoreOwner 来共享 `SharedViewModel`。

---

### 五、注意事项

1. **不要在 Fragment 中用 `by viewModels()`** 来共享，否则每个 Fragment 会得到独立的 ViewModel 实例。
2. **生命周期感知**：使用 `viewLifecycleOwner` 观察 LiveData，避免内存泄漏。
3. **数据类型**：简单数据直接放 LiveData/StateFlow；复杂对象建议使用 `data class` + `StateFlow`。
4. **单向数据流**：推荐 `ViewModel` 暴露不可变的 `StateFlow`，通过函数修改（`sendXXX`）。
5. **Fragment 间导航**：如果使用 Navigation Component，可以结合 `setFragmentResult` / `setFragmentResultListener`，但 ViewModel 更适合**持续共享状态**的场景。

### 六、进阶：使用 Hilt / Dagger 注入

```kotlin
@HiltViewModel
class SharedViewModel @Inject constructor(...) : ViewModel()
```

然后依然使用 `by activityViewModels()`，Hilt 会自动处理。
