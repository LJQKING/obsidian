**Room 增加字段的处理方式（推荐做法）**

在 Android Room 中直接给 `@Entity` 类增加字段，会导致数据库 schema 版本不匹配，App 启动时崩溃。因此**必须进行数据库迁移（Migration）**。

### 1. 修改 Entity 类

```kotlin
@Entity(tableName = "rooms")
data class RoomEntity(
    @PrimaryKey val id: Long,
    val name: String,
    val description: String?,
    
    // 新增字段示例
    val createTime: Long = System.currentTimeMillis(),   // 新增字段
    val isActive: Boolean = true,                        // 新增字段
    val extraData: String? = null                        // 可空字段
)
```

**注意**：
- 新增字段**必须**提供默认值（`=`），否则 Room 无法自动生成 schema。
- 如果是 `String?`、`Int?` 等可空类型，默认值可以是 `null`。

### 2. 更新 Database 版本

```kotlin
@Database(
    entities = [RoomEntity::class, ...],
    version = 2,           // ← 版本号 +1
    exportSchema = true
)
abstract class AppDatabase : RoomDatabase() { ... }
```

### 3. 创建 Migration（最重要）

```kotlin
val MIGRATION_1_2 = object : Migration(1, 2) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("""
            ALTER TABLE rooms 
            ADD COLUMN createTime INTEGER NOT NULL DEFAULT ${System.currentTimeMillis()}
        """.trimIndent())
        
        db.execSQL("""
            ALTER TABLE rooms 
            ADD COLUMN isActive INTEGER NOT NULL DEFAULT 1
        """.trimIndent())
        
        db.execSQL("""
            ALTER TABLE rooms 
            ADD COLUMN extraData TEXT
        """.trimIndent())
    }
}
```

**类型对应关系**：
- `Int` / `Long` → `INTEGER`
- `String` → `TEXT`
- `Boolean` → `INTEGER`（0/1）
- `Double` / `Float` → `REAL`

### 4. 在 Database Builder 中添加 Migration

```kotlin
val db = Room.databaseBuilder(
    context,
    AppDatabase::class.java,
    "app.db"
)
    .addMigrations(MIGRATION_1_2)   // 支持多个：.addMigrations(M1, M2, M3)
    .build()
```

### 推荐的完整写法（更清晰）

```kotlin
companion object {
    private val MIGRATION_1_2 = object : Migration(1, 2) {
        override fun migrate(db: SupportSQLiteDatabase) {
            db.execSQL("ALTER TABLE rooms ADD COLUMN createTime INTEGER NOT NULL DEFAULT 0")
            db.execSQL("ALTER TABLE rooms ADD COLUMN isActive INTEGER NOT NULL DEFAULT 1")
            db.execSQL("ALTER TABLE rooms ADD COLUMN extraData TEXT")
        }
    }

    fun getDatabase(context: Context): AppDatabase {
        return Room.databaseBuilder(context, AppDatabase::class.java, "app.db")
            .addMigrations(MIGRATION_1_2)
            // .fallbackToDestructiveMigration() // 仅开发阶段使用，正式版不要加
            .build()
    }
}
```

### 其他注意事项

1. **多个字段**：建议把所有 ALTER TABLE 语句写在同一个 `Migration` 里。
2. **开发阶段**：可以临时使用 `.fallbackToDestructiveMigration()`，但**正式发布前必须删除**，否则用户数据会丢失。
3. **默认值**：如果字段需要当前时间等动态默认值，在 Migration 中用 `0` 或 `''` 占位，后续在代码中处理。
4. **索引/外键**：如果新增字段需要索引，Migration 中也要加上 `CREATE INDEX` 语句。
5. **测试**：升级后一定要在不同版本的数据库上测试迁移是否成功。

### 进阶建议

- 使用 **Room Schema Export**（`exportSchema = true`）配合版本控制，能更好地追踪 schema 变化。
- 大版本升级时，推荐使用 **AutoMigration**（Room 2.6.0+）简化写法：

```kotlin
@AutoMigration(from = 1, to = 2)
class MyAutoMigration
```

然后在 `@Database` 中声明 `autoMigrations = [AutoMigration(1, 2)]`。
