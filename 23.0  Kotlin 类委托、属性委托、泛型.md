**Kotlin 类委托、属性委托、泛型详解**

以下是 Kotlin 中这三个重要特性的系统性说明，包含语法、原理、示例和使用场景。

### 1. 泛型 (Generics)

Kotlin 的泛型与 Java 类似，但更简洁、支持**型变**（variance）和**reified 类型**。

#### 基本语法
```kotlin
// 泛型类
class Box<T>(val value: T)

// 泛型函数
fun <T> printItem(item: T) {
    println(item)
}

// 泛型约束
fun <T : Number> sum(a: T, b: T): Double = a.toDouble() + b.toDouble()

// 多个约束
fun <T> sort(list: List<T>) where T : Comparable<T>, T : Any { ... }
```

#### 型变（Variance）
- **out**（协变）：生产者（Producer），只能读取（`out T`）
- **in**（逆变）：消费者（Consumer），只能写入（`in T`）
- 默认不变（invariant）

```kotlin
// 协变示例
interface Producer<out T> {
    fun produce(): T
}

// 逆变示例
interface Consumer<in T> {
    fun consume(item: T)
}

// 使用
val strProducer: Producer<String> = ...
val anyProducer: Producer<Any> = strProducer  // 合法（协变）
```

#### reified（具体化）—— 内联函数特有
```kotlin
inline fun <reified T> isTypeOf(obj: Any): Boolean = obj is T

// 使用
if (isTypeOf<String>("hello")) { ... }
```

**常见应用**：集合（`List<T>`）、`Result<T>`、依赖注入等。

### 2. 类委托 (Class Delegation)

使用 `by` 关键字将一个接口的实现委托给另一个对象，实现**组合优于继承**。

#### 语法
```kotlin
interface Base {
    fun print()
}

class BaseImpl(val x: Int) : Base {
    override fun print() { println(x) }
}

// 委托
class Derived(b: Base) : Base by b

// 使用
val base = BaseImpl(10)
val derived = Derived(base)
derived.print()  // 输出 10
```

#### 委托多个接口 + 覆盖部分方法
```kotlin
interface A { fun foo() }
interface B { fun bar() }

class AImpl : A { override fun foo() = println("A") }
class BImpl : B { override fun bar() = println("B") }

class Composite(a: A, b: B) : A by a, B by b {
    override fun foo() { println("Composite foo") }  // 可覆盖
}
```

**使用场景**：
- 装饰器模式
- 代理模式
- 避免深层继承
- Kotlin 标准库中 `Delegates`、`observable` 等都基于委托实现

### 3. 属性委托 (Property Delegation)

使用 `by` 关键字把属性的 `get`/`set` 逻辑委托给一个**委托对象**。

#### 内置委托

```kotlin
import kotlin.properties.Delegates

class Example {
    // 1. 延迟初始化
    val lazyValue: String by lazy {
        println("computed!")
        "Hello"
    }

    // 2. 可观察属性
    var observableProp: Int by Delegates.observable(0) { prop, old, new ->
        println("$old -> $new")
    }

    // 3. 可否决属性（vetoable）
    var vetoableProp: Int by Delegates.vetoable(0) { _, old, new ->
        new > old  // 只有新值更大时才允许修改
    }

    // 4. 映射委托（Map / MutableMap）
    val map = mutableMapOf<String, Any?>()
    var name: String by map
    var age: Int by map
}
```

#### 自定义属性委托

需要实现 `ReadOnlyProperty` 或 `ReadWriteProperty`：

```kotlin
import kotlin.reflect.KProperty

class LoggingDelegate<T>(private var value: T) {
    operator fun getValue(thisRef: Any?, property: KProperty<*>): T {
        println("Getting ${property.name} = $value")
        return value
    }

    operator fun setValue(thisRef: Any?, property: KProperty<*>, newValue: T) {
        println("Setting ${property.name} from $value to $newValue")
        value = newValue
    }
}

class User {
    var email: String by LoggingDelegate("initial@example.com")
}
```

**委托属性要求**：
- `val` 属性只需 `getValue`
- `var` 属性需要 `getValue` + `setValue`
- 接收者可以是 `thisRef: Any?` 和 `property: KProperty<*>`

### 结合使用示例

```kotlin
// 泛型 + 属性委托 + 类委托综合
class Repository<T> {
    private val cache = mutableMapOf<String, T>()
    
    var data: T? by Delegates.observable(null) { _, _, new ->
        if (new != null) cache["latest"] = new
    }
}

interface DataService<T> {
    fun fetch(): T
}

class CachedService<T>(private val realService: DataService<T>) : DataService<T> by realService {
    private var cache: T? by Delegates.notNull()  // 要求必须初始化
}
```

### 推荐学习顺序

1. **先掌握泛型**（尤其是 `out`/`in` 和 `reified`）
2. **再学属性委托**（`lazy`、`observable` 是日常最常用）
3. **最后类委托**（用于架构层面解耦）
