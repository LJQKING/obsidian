# ViewModel 和 LiveData 在 MVVM 框架中的作用

## ViewModel 的核心作用

### 1. **生命周期管理与数据持久化**

```kotlin
class MyViewModel : ViewModel() {
    // 配置变更（旋转屏幕）时不会被销毁
    private val _data = MutableLiveData<String>()
    val data: LiveData<String> = _data
    
    init {
        // 只初始化一次，即使Activity重建也保留数据
        loadData()
    }
    
    override fun onCleared() {
        // 清理资源
        super.onCleared()
    }
}
```

**原理：**

- ViewModel 持有时间长于 Activity/Fragment
- 在配置变更（屏幕旋转）时 Activity 被销毁重建，但 ViewModel 存活
- 由 ViewModelStore 在 NonConfigurationInstances 中保存，避免数据丢失

### 2. **分离 UI 层和业务逻辑**

```kotlin
// ViewModel 层 - 不持有 View 引用
class UserViewModel : ViewModel() {
    private val repository = UserRepository()
    
    private val _userState = MutableLiveData<UiState>()
    val userState: LiveData<UiState> = _userState
    
    fun loadUser(id: String) {
        viewModelScope.launch {
            try {
                val user = repository.getUser(id)
                _userState.value = UiState.Success(user)
            } catch (e: Exception) {
                _userState.value = UiState.Error(e.message)
            }
        }
    }
}

// Activity 层 - 只关注 UI
class UserActivity : AppCompatActivity() {
    private val viewModel: UserViewModel by viewModels()
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        
        viewModel.userState.observe(this) { state ->
            when (state) {
                is UiState.Success -> showUser(state.user)
                is UiState.Error -> showError(state.message)
                is UiState.Loading -> showLoading()
            }
        }
    }
}
```

## LiveData 的核心作用

### 1. **生命周期感知的数据观察**

```kotlin
class MyViewModel : ViewModel() {
    private val _count = MutableLiveData<Int>(0)
    val count: LiveData<Int> = _count
}

class MyFragment : Fragment() {
    private val viewModel: MyViewModel by viewModels()
    
    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)
        
        // observe() 自动感知 Fragment 生命周期
        // Fragment 进入 DESTROYED 时自动移除观察者，防止内存泄漏
        viewModel.count.observe(viewLifecycleOwner) { count ->
            textView.text = count.toString()
        }
        
        // observeForever() 不感知生命周期，需要手动移除
        // viewModel.count.observeForever { }
    }
}
```

**原理：**

- LiveData 实现 LifecycleObserver 接口
- 在观察者注册时检查生命周期状态
- 只在观察者处于 STARTED 或 RESUMED 状态时发送更新
- 自动在合适时机移除观察者，防止内存泄漏

### 2. **数据变化的粘性通知**

```kotlin
// LiveData 会发送最后一个值给新的观察者
val liveData = MutableLiveData<String>("Hello")

// 1秒后订阅，仍会立即收到 "Hello"
Handler(Looper.getMainLooper()).postDelayed({
    liveData.observe(this) { value ->
        println(value)  // 输出：Hello
    }
}, 1000)
```

**使用场景：**

- 配置变更后恢复 UI 状态
- 新的观察者能立即获得最新数据

## MVVM 中的协作流程

```
┌─────────────────────────────────────────────┐
│              UI Layer (View)                 │
│         (Activity/Fragment)                  │
│                                             │
│  - 观察 ViewModel 中的 LiveData            │
│  - 响应用户交互                             │
│  - 更新 UI 元素                             │
└────────────────┬────────────────────────────┘
                 │ 观察 LiveData
                 │ 调用 ViewModel 方法
                 ▼
┌─────────────────────────────────────────────┐
│         ViewModel Layer                      │
│    (业务逻辑与状态管理)                      │
│                                             │
│  - 持有 LiveData/StateFlow                  │
│  - 协调 Model 和 View 通信                  │
│  - 不持有 View 引用（可测试性）            │
│  - viewModelScope 管理异步任务              │
└────────────────┬────────────────────────────┘
                 │ 调用 Repository
                 ▼
┌─────────────────────────────────────────────┐
│           Model Layer                        │
│    (Repository/Database/Network)            │
│                                             │
│  - 数据访问                                 │
│  - 业务规则实现                             │
└─────────────────────────────────────────────┘
```

## 源码原理细节

### ViewModel 的创建与保存

```kotlin
// ViewModelStore - 保存 ViewModel 实例
class ViewModelStore {
    private val mMap = mutableMapOf<String, ViewModel>()
    
    fun put(key: String, viewModel: ViewModel) {
        val oldViewModel = mMap.put(key, viewModel)
        oldViewModel?.onCleared()
    }
    
    fun get(key: String): ViewModel? = mMap[key]
}

// 配置变更时的保存流程
// 1. Activity.onRetainNonConfigurationInstance() 在销毁前调用
// 2. ViewModelStore 被保存在 NonConfigurationInstances
// 3. 新 Activity 创建时，通过 getLastNonConfigurationInstance() 恢复
```

### LiveData 的观察机制

```kotlin
abstract class LiveData<T> : LifecycleObserver {
    private val mObservers = mutableMapOf<Observer<in T>, LifecycleBoundObserver>()
    
    public fun observe(owner: LifecycleOwner, observer: Observer<in T>) {
        // 1. 包装观察者为生命周期感知对象
        val wrapper = LifecycleBoundObserver(owner.lifecycle, observer)
        
        // 2. 注册为生命周期观察者
        owner.lifecycle.addObserver(wrapper)
        
        // 3. 立即分发最新值（粘性通知）
        if (lifecycle.currentState >= STARTED) {
            observer.onChanged(mValue)
        }
    }
    
    public fun setValue(value: T) {
        mValue = value
        // 只通知处于 STARTED 以上状态的观察者
        dispatchingValue(null)
    }
}
```

## 最佳实践

```kotlin
class UserViewModel : ViewModel() {
    // 1. 数据只读，防止外部修改
    private val _userData = MutableLiveData<User>()
    val userData: LiveData<User> = _userData
    
    // 2. 使用 LiveData 变换
    val userNameDisplay: LiveData<String> = userData.map { user ->
        "${user.firstName} ${user.lastName}"
    }
    
    // 3. 使用 viewModelScope 自动取消协程
    fun loadUser(id: String) {
        viewModelScope.launch {
            val user = userRepository.getUser(id)
            _userData.value = user
        }
    }
    
    // 4. StateFlow 是 LiveData 的现代替代品
    private val _uiState = MutableStateFlow<UiState>(UiState.Idle)
    val uiState: StateFlow<UiState> = _uiState.asStateFlow()
}
```

## ViewModel vs StateFlow

|特性|LiveData|StateFlow|
|---|---|---|
|生命周期感知|✅ 内置|❌ 需手动处理|
|粘性通知|✅|✅|
|线程安全|✅|✅|
|Kotlin Flow|❌|✅|
|现代推荐|中等|✅ 推荐|

**建议：** 新项目优先使用 StateFlow + collectAsState，但 LiveData 在 Android Studio Templates 中仍广泛使用。
