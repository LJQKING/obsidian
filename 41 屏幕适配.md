**Android 屏幕适配的核心原理**是解决设备碎片化问题（不同屏幕尺寸、分辨率、像素密度）。Android 系统使用 **dp（density-independent pixel）** 和 **sp（scale-independent pixel）** 作为标准单位，结合资源限定符实现基本适配，但实际开发中仍需额外方案处理复杂场景。

### 1. 基础原理
- **密度（density）与转换公式**：
  - `px = dp * density`
  - `density = dpi / 160`（160 是基准 mdpi 密度）
  - 布局中的 dp 值最终通过 `TypedValue.applyDimension` 转为 px。

- **主要问题**：不同设备 density 不同，导致同一 dp 值在不同屏幕上 px 差异大；宽高比不同也容易变形。

- **官方推荐做法**（始终优先）：
  - 使用 `ConstraintLayout`、`Guideline` 等灵活布局。
  - 避免硬编码 px 或固定尺寸。
  - 使用 `android:resizeableActivity="true"` 支持多窗口/折叠屏/大屏。
  - 提供替代资源：`layout-sw600dp`、`values-w820dp` 等 **smallestWidth (sw)** 限定符、`drawable-mdpi/hdpi/xhdpi` 等。
  - Jetpack Compose 的自适应布局（推荐现代项目）。

### 2. 常见手动适配方案
- **smallestWidth (sw) 限定符**：根据最小宽度 dp 值提供不同 dimens.xml，稳定且官方友好。
- **宽高限定符**：`layout-1280x720` 等，但维护成本高，不推荐全量使用。
- **修改 density（今日头条方案核心）**：在 Activity onCreate 前动态修改 `DisplayMetrics.density`，让设计稿宽度（如 360dp）在所有设备上对应实际屏幕宽度。

示例手动代码（今日头条思路）：

```kotlin
fun setCustomDensity(activity: Activity, application: Application, designWidthDp: Int = 360) {
    val appDisplayMetrics = application.resources.displayMetrics
    val targetDensity = appDisplayMetrics.widthPixels / designWidthDp.toFloat()
    
    appDisplayMetrics.density = targetDensity
    appDisplayMetrics.scaledDensity = targetDensity
    appDisplayMetrics.densityDpi = (targetDensity * 160).toInt()
    
    val activityDisplayMetrics = activity.resources.displayMetrics
    activityDisplayMetrics.density = targetDensity
    activityDisplayMetrics.scaledDensity = targetDensity
    activityDisplayMetrics.densityDpi = (targetDensity * 160).toInt()
}
```

在 Application 或 BaseActivity 中调用，并处理 configuration 变化。

### 3. 推荐第三方库：**AndroidAutoSize**（最常用）
这是 **今日头条方案的终极封装版**，低成本、侵入小、支持自定义，维护活跃。

**优点**：
- 自动适配（基于 Activity/Fragment 生命周期）。
- 支持取消特定页面适配、子单位（pt/subunits）、第三方库适配。
- 配置简单，只需设计稿尺寸。

**集成步骤**：

1. **添加依赖**（最新版本查 GitHub）：
   ```gradle
   implementation 'me.jessyan:autosize:1.2.1'  // 或更高版本
   ```

2. **AndroidManifest.xml 配置设计稿尺寸**（推荐 360dp 宽度）：
   ```xml
   <application>
       <meta-data
           android:name="design_width_in_dp"
           android:value="360"/>
       <meta-data
           android:name="design_height_in_dp"
           android:value="640"/>
   </application>
   ```

3. **Application 初始化**：
   ```kotlin
   class MyApplication : Application() {
       override fun onCreate() {
           super.onCreate()
           AutoSizeConfig.getInstance()
               .setLog(true)  // 调试日志
               .setUseDeviceSize(true)  // 使用设备真实尺寸
               // 更多配置...
       }
   }
   ```

4. **自定义控制**（可选）：
   - 实现 `CustomAdapt` 接口指定宽/高适配。
   - 实现 `CancelAdapt` 取消适配。
   ```kotlin
   class MainActivity : AppCompatActivity(), CustomAdapt {
       override fun isBaseOnWidth(): Boolean = true  // true: 基于宽度
       override fun getSizeInDp(): Float = 360f
   }
   ```

详细使用见官方 GitHub：https://github.com/JessYanCoding/AndroidAutoSize。

**其他第三方**：
- **ScreenAdapter** 或 **DensityHelper**：简单工具类，一行代码适配。
- **AndroidUtilCode**（Blankj）：提供 pt 适配方案。
- 对于 Compose 项目，直接用官方 Adaptive Layouts（如 Material 3 Adaptive）。

### 4. 操作建议与最佳实践
1. **设计阶段**：UI 按固定宽度设计稿（如 375pt 或 360dp）出图。
2. **开发阶段**：
   - 优先 ConstraintLayout + 权重/百分比。
   - 文字用 sp，图片用 .9.png 或 VectorDrawable。
   - 测试多设备（包括折叠屏、大屏平板）。
3. **大屏/折叠屏适配**：支持 `resizeableActivity`，使用 WindowManager API 或 Jetpack Window 库监听窗口大小变化。
4. **注意事项**：
   - 修改 density 会全局影响，注意 WebView、第三方库、Dialog/Fragment 适配。
   - Android 12+ 隐私/配置变化处理 configuration。
   - 预览时在 Android Studio 创建对应 dp 虚拟设备。

**总结推荐**：
- 新项目：**AndroidAutoSize** + 官方灵活布局。
- 老项目迁移：逐步替换为 AutoSize 或 sw 限定符。
- 极致要求：结合 Jetpack Compose。
