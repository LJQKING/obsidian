# ViewGroup 和 View 的区别及关系

在 Android UI 体系中：

```text
Object
 └── View
      ├── TextView
      ├── Button
      ├── ImageView
      └── ViewGroup
           ├── LinearLayout
           ├── RelativeLayout
           ├── FrameLayout
           ├── ConstraintLayout
           ├── RecyclerView
           └── ScrollView
```

可以看到：

> **ViewGroup 本质上也是 View 的子类。**

因此：

- 所有 ViewGroup 都是 View
    
- 但不是所有 View 都是 ViewGroup
    

---

# 一、View是什么

View 是 Android 中最基本的 UI 组件。

负责：

- 显示内容
    
- 接收用户事件
    
- 绘制界面
    

例如：

```java
TextView
Button
EditText
ImageView
```

继承关系：

```java
public class View
```

例如：

```java
TextView tv = findViewById(R.id.tv);

tv.setText("Hello");
tv.setOnClickListener(v -> {
    Log.d("TAG", "点击");
});
```

View 的核心职责：

```text
测量(Measure)
    ↓
布局(Layout)
    ↓
绘制(Draw)
    ↓
事件处理(Touch)
```

---

# 二、ViewGroup是什么

ViewGroup 是一个特殊的 View。

除了自己能显示外：

还可以管理子 View。

例如：

```java
LinearLayout
RelativeLayout
FrameLayout
ConstraintLayout
RecyclerView
```

源码：

```java
public abstract class ViewGroup extends View
```

内部维护：

```java
private View[] mChildren;
```

示意：

```text
LinearLayout(ViewGroup)

 ├── TextView
 ├── Button
 └── ImageView
```

代码：

```java
LinearLayout layout = findViewById(R.id.root);

Button btn = new Button(this);

layout.addView(btn);
```

---

# 三、View 和 ViewGroup 的区别

|对比项|View|ViewGroup|
|---|---|---|
|是否能显示|√|√|
|是否能接收事件|√|√|
|是否能包含子View|×|√|
|是否负责布局|×|√|
|是否能addView|×|√|
|是否参与绘制|√|√|

---

# 四、ViewGroup为什么继承View

因为布局本身也是一个View。

例如：

```xml
<LinearLayout>

</LinearLayout>
```

这个布局：

- 有宽高
    
- 有背景
    
- 可以点击
    
- 可以绘制
    

例如：

```java
layout.setBackgroundColor(Color.RED);

layout.setOnClickListener(...);
```

所以：

```java
ViewGroup extends View
```

是非常合理的设计。

# 六、Measure阶段区别

## View

测量自己

```java
onMeasure()
```

例如：

```java
TextView
```

只计算：

```text
自己的宽高
```

---

## ViewGroup

不仅测量自己

还要测量所有子View

源码：

```java
@Override
protected void onMeasure(
        int widthMeasureSpec,
        int heightMeasureSpec) {

    measureChildren(
        widthMeasureSpec,
        heightMeasureSpec);

    setMeasuredDimension(
        width,
        height);
}
```

例如：

```java
LinearLayout
```

需要：

```text
先测量孩子
再计算自己
```

---

# 七、Layout阶段区别

## View

确定自己的位置

```java
layout(
    left,
    top,
    right,
    bottom
);
```

---

## ViewGroup

负责摆放所有孩子

例如：

```java
@Override
protected void onLayout(
        boolean changed,
        int l,
        int t,
        int r,
        int b) {

    child.layout(...);
}
```

LinearLayout：

```text
child1
child2
child3
```

竖直排列：

```text
top = 0

child1
↓
child2
↓
child3
```

---

# 八、Draw阶段区别

## View

绘制自己

源码：

```java
draw()
```

执行：

```java
onDraw()
```

例如：

```java
TextView
ImageView
```

---

## ViewGroup

默认：

```java
不执行onDraw()
```

因为主要负责管理子View。

流程：

```java
draw()
    ↓
dispatchDraw()
    ↓
child.draw()
```

源码：

```java
protected void dispatchDraw(Canvas canvas)
```

作用：

```text
绘制所有子View
```

---

# 九、事件分发区别

这是面试高频。

## View

直接消费事件

```java
onTouchEvent()
```

例如：

```java
Button
```

```java
@Override
public boolean onTouchEvent(
        MotionEvent event) {

    return true;
}
```

---

## ViewGroup

负责分发事件

核心源码：

```java
dispatchTouchEvent()
```

流程：

```text
ViewGroup
     ↓
dispatchTouchEvent
     ↓
onInterceptTouchEvent
     ↓
Child View
     ↓
onTouchEvent
```

例如：

```java
RecyclerView
```

会决定：

```text
事件给自己
还是给Item
```

# 十一、面试标准答案

**View 和 ViewGroup 的关系：**

- View 是 Android UI 的基础类，负责显示和事件处理。
    
- ViewGroup 继承自 View，是一种特殊的 View。
    
- ViewGroup 除了拥有 View 的能力外，还可以包含和管理多个子 View。
    
- Android 界面本质上是一棵 View Tree，根节点通常是 DecorView，各种 Layout（LinearLayout、ConstraintLayout 等）属于 ViewGroup，Button、TextView 等属于普通 View。
    

**核心区别：**

|功能|View|ViewGroup|
|---|---|---|
|显示内容|√|√|
|处理点击事件|√|√|
|包含子View|×|√|
|管理布局|×|√|
|事件分发|接收事件|分发事件|
|典型类|TextView、Button|LinearLayout、ConstraintLayout、RecyclerView|

一句话总结：

> **View 负责“显示自己”，ViewGroup 负责“管理和显示一组 View”，Android 整个界面实际上就是由 View 和 ViewGroup 组成的一棵树状结构。**
