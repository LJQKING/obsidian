# synchronized 实现逻辑详解

## 1. 字节码层面

```java
public void test() {
    synchronized(this) {
        System.out.println("hello");
    }
}
```

编译后的字节码：

```
 0: aload_0
 1: dup
 2: astore_1
 3: monitorenter          // 进入monitor
 4: getstatic     #2     // System.out
 7: ldc           #3     // "hello"
 9: invokevirtual #4     // println
12: aload_1
13: monitorexit           // 退出monitor
14: goto          22
17: astore_2
18: aload_1
19: monitorexit           // 异常时也要释放
20: athrow
21: athrow
22: return
```

**关键指令：**

- `monitorenter`：获取对象的 monitor（互斥锁）
- `monitorexit`：释放对象的 monitor

## 2. 对象头结构（Mark Word）

Java 对象内存布局：

```
┌─────────────────────────────────┐
│    对象头 (Object Header)        │
├─────────────────────────┬───────┤
│   Mark Word (8字节)     │ Klass │
├─────────────────────────┤ Ptr   │
│ ……                      │ (8字) │
├─────────────────────────┴───────┤
│        实例数据                   │
├─────────────────────────────────┤
│        对齐填充                   │
└─────────────────────────────────┘
```

**Mark Word 结构（64位）：**

|锁状态|Mark Word 结构|偏向锁标记|锁标记|
|---|---|---|---|
|无锁|`hash[25]|age[4]|biased_lock[1]|
|偏向锁|`threadId[54] \| epoch[2] \| unused[1] \| biased_lock[1] \| lock[2]`|1|01|
|轻量级锁|`ptr_to_lock_record[62] \| lock[2]`|-|00|
|重量级锁|`ptr_to_monitor[62] \| lock[2]`|-|10|

## 3. 锁升级过程

### ① 无锁 → 偏向锁

```
条件：
- 对象创建时进入偏向锁模式（-XX:+UseBiasedLocking）
- 第一个线程获取锁时

操作：
- CAS 将当前线程 ID 写入 Mark Word
- 该线程再次获取锁时无需任何操作（快速路径）
```

**偏向锁代码路径（简化）：**

```cpp
// hotspot/src/share/vm/runtime/synchronizer.cpp
void ObjectSynchronizer::slow_enter(
    Handle obj, BasicLock* lock, TRAPS) {
  
  markOop mark = obj->mark();
  
  // 尝试偏向锁
  if (mark->has_bias_pattern()) {
    // 检查偏向线程
    Thread* biased_thread = ...;
    if (biased_thread == current_thread) {
      // 同一线程重入，直接返回
      return;
    }
  }
  
  // 偏向锁冲突 → 升级到轻量级锁
  inflate(obj);
}
```

### ② 偏向锁 → 轻量级锁

**触发条件：**

- 第二个线程尝试获取锁
- 当前线程已获取偏向锁

**升级过程：**

```cpp
markOop ObjectMonitor::deflate_monitor() {
  // 1. 撤销偏向
  markOop mark = obj->mark();
  if (mark->has_bias_pattern()) {
    // CAS 清除偏向锁标记
    obj->cas_set_mark(unbiased_mark, mark);
  }
  
  // 2. 创建 LockRecord（在线程栈上）
  BasicLock* lock = new BasicLock();
  markOop displaced_mark = obj->mark();
  lock->set_displaced_header(displaced_mark);
  
  // 3. CAS 将 Mark Word 设置为 LockRecord 指针
  if (!obj->cas_set_mark((markOop)lock, displaced_mark)) {
    // CAS 失败 → 再升级到重量级锁
  }
}
```

### ③ 轻量级锁 → 重量级锁

**触发条件：**

- 自旋超过阈值（默认10次）
- 其他线程在自旋等待时又有新线程来竞争

**重量级锁（ObjectMonitor）：**

```cpp
ObjectMonitor {
  volatile markOop _header;           // 原始 Mark Word
  void* _object;                       // 监视的对象
  
  ObjectWaiter* _owner;                // 当前持有者线程
  ObjectWaiter* _EntryList;            // 等待队列
  ObjectWaiter* _WaitSet;              // wait() 等待队列
  
  volatile int _waiters;
  volatile int _recursions;            // 重入计数
}
```

## 4. 完整获取过程

```cpp
void ObjectSynchronizer::enter(
    Handle obj, BasicLock* lock, TRAPS) {
  
  // 阶段1：快速路径 - 尝试 CAS
  markOop mark = obj->mark();
  if (mark->is_unlocked()) {
    // 对象处于无锁状态
    BasicLock* displaced = mark;
    if (obj->cas_set_mark((markOop)lock, mark) == mark) {
      lock->set_displaced_header(displaced);
      return;  // ✓ 成功
    }
  }
  
  // 阶段2：重入检测
  if (mark == (markOop)lock) {
    // 当前线程已持有轻量级锁
    lock->set_displaced_header(NULL);
    return;  // ✓ 成功重入
  }
  
  // 阶段3：升级到重量级锁
  ObjectMonitor* monitor = inflate(obj);
  monitor->enter_i(THREAD);
}
```

## 5. 释放过程

```cpp
void ObjectSynchronizer::exit(
    Handle obj, BasicLock* lock, TRAPS) {
  
  markOop mark = obj->mark();
  
  if (mark == (markOop)lock) {
    // 轻量级锁
    markOop displaced = lock->displaced_header();
    if (displaced == NULL) {
      // 重入，仅减少计数
      return;
    }
    
    // CAS 恢复原始 Mark Word
    if (!obj->cas_set_mark(displaced, mark)) {
      // CAS 失败 → 转为重量级锁处理
      ObjectMonitor* monitor = inflate(obj);
      monitor->exit_i(THREAD);
    }
  } else {
    // 重量级锁
    ObjectMonitor* monitor = (ObjectMonitor*)mark;
    monitor->exit_i(THREAD);
  }
}
```

## 6. 关键优化

### 自旋（Spin-Lock）

```cpp
// 轻量级锁获取失败时自旋
for (int iter = 0; iter < SpinLimit; iter++) {
  // 轻忙轮询，消耗 CPU 但避免上下文切换
  if (obj->mark_shallow() != (markOop)lock) {
    goto try_again;
  }
}
// 自旋超过阈值，升级到重量级锁
```

### 适应性自旋（Adaptive Spinning）

```cpp
// 根据之前的自旋成功率动态调整
if (spin_success_rate > THRESHOLD) {
  increase_spin_limit();
} else {
  decrease_spin_limit();
}
```

## 7. 性能对比

|锁类型|获取耗时|释放耗时|场景|
|---|---|---|---|
|无竞争偏向锁|0 CAS|0 CAS|单线程|
|轻量级锁|1 CAS|1 CAS|低竞争|
|重量级锁|系统调用|系统调用|高竞争|

## 8. 相关 JVM 参数

```bash
-XX:+UseBiasedLocking           # 启用偏向锁（默认开启）
-XX:BiasedLockingStartupDelay=0 # 启动延迟（ms）
-XX:BiasedLockingBulkRebiasThreshold=20
-XX:BiasedLockingBulkRevokeThreshold=40
-XX:-UseBiasedLocking           # 禁用偏向锁

-XX:PreBlockSpin=10             # 轻量级锁自旋次数
-XX:+UseSpinning                # 启用自旋
```
