蓝牙（尤其是 BLE）发送大数据时，经常会遇到 **分包发送（Packet Fragmentation）** 和 **分包回复（Packet Reassembly）** 的问题。

因为 BLE 单包大小有限：

- BLE4.0 默认 MTU = 23
    
- 可用数据长度 = 20 Byte
    
- BLE5.0 可以协商 MTU（247、512等）
    

例如发送 100 字节数据：

```text
总数据:
01 02 03 ... 64

分成：

包1：20字节
包2：20字节
包3：20字节
包4：20字节
包5：20字节
```

# 一、工业级协议设计

建议每包增加协议头：

```text
┌────┬────┬──────┬──────┬─────────┬────┐
│SOF │CMD │Total │Index │ Data    │CRC │
├────┼────┼──────┼──────┼─────────┼────┤
│AA55│01  │ 05   │ 01   │ xxxx    │xx  │
└────┴────┴──────┴──────┴─────────┴────┘
```

字段说明：

```text
SOF     帧头
CMD     命令
Total   总包数
Index   当前包序号
Data    数据
CRC     校验
```

例如：

```text
总数据长度 = 80字节

Total = 4

包1:
AA55 01 04 01 DATA CRC

包2:
AA55 01 04 02 DATA CRC

包3:
AA55 01 04 03 DATA CRC

包4:
AA55 01 04 04 DATA CRC
```

# 二、Android发送分包

## 分包工具

```java
public static List<byte[]> splitPacket(byte[] data, int packetSize) {

    List<byte[]> packets = new ArrayList<>();

    int total = (data.length + packetSize - 1) / packetSize;

    for (int i = 0; i < total; i++) {

        int start = i * packetSize;

        int len = Math.min(packetSize,
                data.length - start);

        byte[] packet = new byte[len];

        System.arraycopy(data,
                start,
                packet,
                0,
                len);

        packets.add(packet);
    }

    return packets;
}
```

发送：

```java
List<byte[]> packets =
        splitPacket(data, 20);

sendNext(packets, 0);
```

# 三、逐包发送

BLE不能连续疯狂write：

```java
private void sendNext(
        List<byte[]> packets,
        int index) {

    if(index >= packets.size()){
        return;
    }

    bluetoothGattCharacteristic
            .setValue(packets.get(index));

    bluetoothGatt.writeCharacteristic(
            bluetoothGattCharacteristic);
}
```


```java
ByteArrayOutputStream buffer =
        new ByteArrayOutputStream();
```

接收：

```java

        BluetoothGatt gatt,
        BluetoothGattCharacteristic characteristic) {

    byte[] packet =
            characteristic.getValue();
```

# 面试高频问题

### BLE为什么要分包？

因为 BLE 单次传输受 MTU 限制：

```text
默认MTU=23
有效数据=20字节
```

---

### 为什么不能循环write发送？

```java
for(...) {
    writeCharacteristic(...)
}
```

这样会导致：

```text
GATT_BUSY
丢包
覆盖发送
```

必须：

```text
Write Success
      ↓
发送下一包
```

---

### 如何提高BLE传输速度？

1. MTU协商
    

```java
gatt.requestMtu(247);
```

2. 调整连接优先级
    

```java
gatt.requestConnectionPriority(
    BluetoothGatt.CONNECTION_PRIORITY_HIGH);
```

3. Write Without Response
    

```java
WRITE_TYPE_NO_RESPONSE
```

4. 多包流水发送（设备支持时）
    

```text
窗口发送
ACK确认
```

这是实际项目（OTA升级、固件升级、人脸设备、工业蓝牙设备、智能门锁等）中常用的 BLE 分包传输方案。
