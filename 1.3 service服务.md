# Android Service 是什么？

`Service` 是 Android 四大组件之一（Activity、Service、BroadcastReceiver、ContentProvider）。

主要作用：

- 在后台执行长时间任务
    
- 不提供 UI 界面
    
- 即使 Activity 退出，任务仍可继续执行
    
- 支持跨进程通信（AIDL/Binder）
    

常见场景：

- 音乐播放
    
- 蓝牙通信
    
- 文件下载
    
- GPS定位
    
- IM长连接
    
- 后台数据同步
    

---

# Service 生命周期

## 1. Started Service（启动型）

```java
Intent intent = new Intent(this, DownloadService.class);
startService(intent);
```

生命周期：

```text
startService()
       ↓
onCreate()
       ↓
onStartCommand()
       ↓
运行中...
       ↓
stopSelf() 或 stopService()
       ↓
onDestroy()
```

代码：

```java
public class DownloadService extends Service {

    @Override
    public void onCreate() {
        super.onCreate();
        Log.d("TAG","onCreate");
    }

    @Override
    public int onStartCommand(Intent intent,
                              int flags,
                              int startId) {

        Log.d("TAG","开始下载");

        return START_STICKY;
    }

    @Override
    public void onDestroy() {
        super.onDestroy();
    }

    @Override
    public IBinder onBind(Intent intent) {
        return null;
    }
}
```

---

## 2. Bound Service（绑定型）

客户端与 Service 建立连接。

```java
bindService()
```

生命周期：

```text
bindService()
      ↓
onCreate()
      ↓
onBind()
      ↓
客户端调用
      ↓
unbindService()
      ↓
onUnbind()
      ↓
onDestroy()
```

# 本地 Binder 通信

## Service

```java
public class MusicService extends Service {

    private final MyBinder binder = new MyBinder();

    class MyBinder extends Binder {

        public void play() {
            Log.d("TAG","播放音乐");
        }
    }

    @Override
    public IBinder onBind(Intent intent) {
        return binder;
    }
}
```

---

## Activity

```java
private MusicService.MyBinder binder;

private ServiceConnection connection =
        new ServiceConnection() {

    @Override
    public void onServiceConnected(
            ComponentName name,
            IBinder service) {

        binder = (MusicService.MyBinder) service;

        binder.play();
    }

    @Override
    public void onServiceDisconnected(
            ComponentName name) {

    }
};
```

绑定：

```java
Intent intent =
        new Intent(this, MusicService.class);

bindService(
        intent,
        connection,
        BIND_AUTO_CREATE);
```

# 前台 Service（Foreground Service）

Android 8.0后后台 Service 限制非常严格。

例如：

- 音乐播放
    
- 导航
    
- 蓝牙设备连接
    
- 健身记录
    

必须使用前台服务。

---

## 创建通知

```java
Notification notification =
        new NotificationCompat.Builder(this,
                "music")
                .setContentTitle("音乐播放")
                .setSmallIcon(R.mipmap.ic_launcher)
                .build();

startForeground(1, notification);
```

---

## AndroidManifest

```xml
<service
    android:name=".MusicService"
    android:foregroundServiceType="mediaPlayback"/>
```

---

启动：

```java
ContextCompat.startForegroundService(
        this,
        intent);
```

# Service源码原理

很多面试官喜欢问：

> startService 到底发生了什么？

# startService源码流程

```text
Activity
   ↓
ContextImpl.startService()
   ↓
ActivityManagerService(AMS)
   ↓ Binder IPC
ActiveServices
   ↓
创建 ServiceRecord
   ↓
ApplicationThread
   ↓ Binder 回调
ActivityThread
   ↓
handleCreateService()
   ↓
反射创建 Service
   ↓
onCreate()
   ↓
onStartCommand()
```

---

# 源码分析

## ContextImpl

```java
@Override
public ComponentName startService(
        Intent service) {

    return mMainThread
            .getApplicationThread()
            .scheduleCreateService();
}
```

实际上会通过 AMS。

---

## AMS

系统服务：

```java
ActivityManagerService
```

处理：

```java
startService()
```

AMS中：

```java
ActiveServices.startServiceLocked()
```

创建：

```java
ServiceRecord
```


```java
public void handleCreateService(
```

# Service 与 Activity 区别

|项目|Activity|Service|
|---|---|---|
|UI|有|无|
|生命周期|短|长|
|用户可见|是|否|
|后台任务|不适合|适合|
|Binder通信|间接|天然支持|

# Service 与 Thread 区别

很多人误以为：

```text
Service = 后台线程
```

实际上错误。

Service 默认运行在：

```java
Main Thread(UI线程)
```

验证：

```java
Log.d("TAG",
Thread.currentThread().getName());
```

输出：

```text
main
```

因此耗时任务必须开启线程：

```java
@Override
public int onStartCommand(...) {

    new Thread(() -> {

        download();

    }).start();

    return START_STICKY;
}
```

或者：

```java
CoroutineScope(Dispatchers.IO)
```

# 蓝牙项目中的 Service

你的蓝牙项目（RxAndroidBle）通常会这样设计：

```text
MainActivity
      ↓
BluetoothService
      ↓
RxAndroidBle
      ↓
BLE Device
```

Service负责：

- 扫描设备
    
- 自动重连
    
- 保持连接
    
- 发送命令
    
- 接收通知
    

Activity负责：

- 显示UI
    

这样即使页面关闭：

```text
Service仍然保持蓝牙连接
```

这是工业级 BLE App 的标准架构。

# Service 面试高频题

### 1. Service运行在哪个线程？

主线程（UI线程）。

---

### 2. startService和bindService区别？

- startService：启动任务
    
- bindService：提供接口调用
    

---

### 3. Service为什么能跨进程？

因为底层基于 Binder IPC。

---

### 4. Android 8.0后为什么启动不了后台Service？

后台启动限制，需要使用前台服务（Foreground Service）。

---

### 5. Service如何避免被杀？

- 前台服务
    
- START_STICKY
    
- JobScheduler
    
- WorkManager
    

---

### 6. Service创建过程？

```text
startService
 ↓
AMS
 ↓
ActiveServices
 ↓
ApplicationThread
 ↓
ActivityThread
 ↓
反射创建Service
 ↓
onCreate
 ↓
onStartCommand
```

这是 Android Framework 中 Service 启动的核心源码链路。
