## Handler 和 HandlerThread 的区别

这是 Android 面试中非常经典的问题。

### 一句话区别

- **Handler**：消息发送和处理工具，本身不是线程。
    
- **HandlerThread**：带有 `Looper` 和 `MessageQueue` 的线程，可以处理 Handler 消息。
    

---

## 1. Handler 是什么？

Handler 用于：

- 发送消息（Message）
    
- 发送任务（Runnable）
    
- 切换线程
    
- 处理消息
    

例如：

```java
Handler mainHandler = new Handler(Looper.getMainLooper());

mainHandler.post(() -> {
    textView.setText("更新UI");
});
```

这里的 Handler：

```text
Handler
   ↓
MessageQueue
   ↓
Looper
   ↓
Thread
```

实际上 Handler 只是：

```text
消息的发送者和处理者
```

它自己不是线程。

---

## 2. HandlerThread 是什么？

HandlerThread 继承自 Thread：

```java
public class HandlerThread extends Thread
```

但它比普通 Thread 多了：

```text
Looper
MessageQueue
```

源码核心：

```java
@Override
public void run() {

    Looper.prepare();

    synchronized (this) {
        mLooper = Looper.myLooper();
        notifyAll();
    }

    Looper.loop();
}
```

相当于：

```java
Thread thread = new Thread() {
    @Override
    public void run() {

        Looper.prepare();

        Handler handler = new Handler();

        Looper.loop();
    }
};
```

Google 帮你封装好了。

---

## 3. 核心区别

|对比项|Handler|HandlerThread|
|---|---|---|
|是否是线程|❌|✅|
|是否有 Looper|❌|✅|
|是否有 MessageQueue|❌|✅|
|是否能执行任务|依赖 Looper|✅|
|主要作用|发送/处理消息|执行消息循环|
|生命周期|跟随 Looper|跟随线程|

---

## 4. 使用关系

通常二者一起使用：

```java
HandlerThread handlerThread =
        new HandlerThread("WorkThread");

handlerThread.start();

Handler workHandler =
        new Handler(handlerThread.getLooper());
```

结构：

```text
Handler
    ↓
MessageQueue
    ↓
Looper
    ↓
HandlerThread
```

发送任务：

```java
workHandler.post(() -> {
    Log.d("TAG", Thread.currentThread().getName());
});
```

输出：

```text
WorkThread
```

---

## 5. Handler 为什么需要 Looper？

创建 Handler 时：

```java
new Handler()
```

源码：

```java
public Handler() {

    mLooper = Looper.myLooper();

    if (mLooper == null) {
        throw new RuntimeException(
            "Can't create handler inside thread");
    }
}
```

说明：

```text
Handler 必须绑定 Looper
Looper 必须存在 MessageQueue
```

因此：

```java
new Thread(() -> {
    Handler handler = new Handler();
}).start();
```

会直接崩溃：

```text
Can't create handler inside thread
```

因为普通 Thread 没有 Looper。

---

## 6. 为什么 HandlerThread 能直接用？

因为它启动后自动执行：

```java
Looper.prepare();
Looper.loop();
```

所以：

```java
HandlerThread ht = new HandlerThread("BLE");
ht.start();

Handler handler =
        new Handler(ht.getLooper());
```

能够正常工作。

## 7. 实际项目中的应用

### Handler

主线程更新 UI

```java
Handler mainHandler =
        new Handler(Looper.getMainLooper());

mainHandler.post(() -> {
    tv.setText("Success");
});
```

---

### HandlerThread

蓝牙命令队列

```java
HandlerThread bleThread =
        new HandlerThread("BLE");

bleThread.start();

Handler bleHandler =
        new Handler(bleThread.getLooper());

bleHandler.post(() -> {
    sendBleCommand();
});
```

特点：

```text
FIFO
单线程
顺序执行
```

非常适合：

- 蓝牙通信
    
- Socket消息处理
    
- 串口通信
    
- 日志写入
    
- 数据库存储
    

---

## 8. HandlerThread 和线程池区别

### HandlerThread

```text
单线程
串行执行
有消息队列
```

例如：

```text
蓝牙
串口
Socket
```

---

### ThreadPoolExecutor

```text
多线程
并发执行
```

例如：

```text

```

## 面试标准答案

**Handler 是 Android 消息机制中的消息发送和处理工具，本身不是线程，它依赖 Looper 和 MessageQueue 工作。**

**HandlerThread 是一个带有 Looper 和 MessageQueue 的线程，本质上继承自 Thread，在 run() 中自动调用 Looper.prepare() 和 Looper.loop()，因此可以持续处理 Handler 投递的消息。**

**Handler 负责发消息，HandlerThread 负责运行消息循环，两者通常配合使用。HandlerThread 适用于蓝牙、Socket、串口等需要顺序执行任务的场景。**
