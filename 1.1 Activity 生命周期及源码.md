# Activity 生命周期及源码解析（面试高级版）

Activity 是 Android 四大组件中最核心的组件之一。理解生命周期不仅要知道回调顺序，更要理解 **ActivityThread → Instrumentation → Activity** 的调用链。

---

# 一、Activity 生命周期总览

## 正常启动

```text
onCreate()
    ↓
onStart()
    ↓
onResume()
```

此时 Activity 进入前台，可与用户交互。

---

## 按 Home 键

```text
onPause()
    ↓
onStop()
```

Activity进入后台。

---

## 再次回到前台

```text
onRestart()
    ↓
onStart()
    ↓
onResume()
```

---

## 按 Back 键

```text
onPause()
    ↓
onStop()
    ↓
onDestroy()
```

Activity被销毁。

---

## 生命周期图

```text
            onCreate()
                 ↓
             onStart()
                 ↓
             onResume()
                 ↓
      ┌────────用户操作────────┐
      ↓                        ↓
  onPause()                onPause()
      ↓                        ↓
  onResume()              onStop()
                               ↓
                         onRestart()
                               ↓
                          onStart()
                               ↓
                          onResume()

                         或

                         onDestroy()
```

# 二、生命周期详解

---

## 1. onCreate()

Activity 创建时调用。

### 主要工作

```java
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);

    setContentView(R.layout.activity_main);

    initView();
    initData();
}
```

一般用于：

- 加载布局
    
- 初始化View
    
- 初始化ViewModel
    
- 注册监听器
    

---

### 源码

```java
public class Activity {

    protected void onCreate(Bundle savedInstanceState) {

    }
}
```

真正调用位置：

```java
Instrumentation.callActivityOnCreate()
```

---

## 2. onStart()

Activity可见但不可交互。

```java
@Override
protected void onStart() {
    super.onStart();
}
```

例如：

- 注册广播
    
- 注册EventBus
    

---

## 3. onResume()

Activity进入前台。

```java
@Override
protected void onResume() {
    super.onResume();
}
```

此时：

- 获取焦点
    
- 用户可点击界面
    

常用于：

```java
camera.startPreview();
```

```java
locationManager.startLocation();
```

---

## 4. onPause()

Activity失去焦点。

```java
@Override
protected void onPause() {
    super.onPause();
}
```

特点：

- 时间必须短
    
- 下一Activity启动前必须执行完
    

常用于：

```java
video.pause();
```

```java
saveDraft();
```

---

## 5. onStop()

Activity完全不可见。

```java
@Override
protected void onStop() {
    super.onStop();
}
```

常用于：

```java
unregisterReceiver();
```

```java
releaseCamera();
```

---

## 6. onRestart()

从后台重新回到前台。

```java
@Override
protected void onRestart() {
    super.onRestart();
}
```

执行顺序：

```text
onRestart()
↓
onStart()
↓
onResume()
```

---

## 7. onDestroy()

Activity销毁。

```java
@Override
protected void onDestroy() {
    super.onDestroy();
}
```

常用于：

```java
handler.removeCallbacksAndMessages(null);

binding = null;
```

```java
presenter.detachView();
```

---

# 三、异常生命周期

---

## 屏幕旋转

默认：

```text
onPause()
↓
onStop()
↓
onDestroy()

重新创建

onCreate()
↓
onStart()
↓
onResume()
```

因为系统认为配置发生变化。

---

## 配置变化保存

```java
@Override
public void onSaveInstanceState(Bundle outState) {

    outState.putString("name","Tom");

    super.onSaveInstanceState(outState);
}
```

恢复：

```java
@Override
protected void onCreate(Bundle savedInstanceState) {

    if(savedInstanceState != null){

        String name =
            savedInstanceState.getString("name");
    }
}
```

# Activity.startActivity()

开发者调用：

---

## Activity.java

```java
public void startActivity(Intent intent) {

    Instrumentation.ActivityResult ar =
        mInstrumentation.execStartActivity(
            this,
            mMainThread.getApplicationThread(),
            mToken,
            this,
            intent,
            -1,
            options);
}
```

# 七、面试高频问题

## 1. onStart和onResume区别？

### onStart

- 页面可见
    
- 不可交互
    

### onResume

- 页面可见
    
- 可交互
    
- 获得焦点
    

---

## 2. onPause和onStop区别？

### onPause

失去焦点。

### onStop

完全不可见。

---

## 3. 横竖屏切换生命周期？

默认：

```text
onPause
onStop
onDestroy

onCreate
onStart
onResume
```

---

## 4. Activity为什么不能new？

错误：

```java
MainActivity activity =
    new MainActivity();
```

原因：

Activity必须经过系统创建：

```text
ActivityThread
↓
Instrumentation
↓
attach()
↓
PhoneWindow
↓
生命周期
```

直接 new：

- Context为空
    
- Window为空
    
- 生命周期不执行
    

必然异常。

---
