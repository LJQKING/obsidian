# Android BroadcastReceiver 详解

**BroadcastReceiver** 是 Android 中用于接收和处理系统或应用广播的组件。它是一种异步消息传递机制，允许应用程序监听全局的广播事件。

可以这样理解：它类似于一个"广播电台接收器"，系统或其他应用发出广播，BroadcastReceiver 就能接收并做出相应的反应。

## 能干什么

- 监听系统事件（手机重启、电池电量低、网络连接变化等）
- 接收来自其他应用的自定义广播
- 在后台执行耗时操作（有限制，不超过 10 秒）
- 启动服务或 Activity
- 更新 UI（如果接收器在 foreground）

## 实际例子

### 1. **监听手机启动完成**（静态注册）

首先在 `AndroidManifest.xml` 中声明：

```xml
<manifest ...>
    <uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
    
    <application>
        <receiver
            android:name=".BootCompletedReceiver"
            android:exported="false">
            <intent-filter>
                <action android:name="android.intent.action.BOOT_COMPLETED" />
            </intent-filter>
        </receiver>
    </application>
</manifest>
```

创建接收器类：

```kotlin
class BootCompletedReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context?, intent: Intent?) {
        if (intent?.action == Intent.ACTION_BOOT_COMPLETED) {
            // 手机启动完成，可以启动服务或做其他初始化
            val serviceIntent = Intent(context, MyBackgroundService::class.java)
            context?.startService(serviceIntent)
            
            Toast.makeText(context, "手机已启动", Toast.LENGTH_SHORT).show()
        }
    }
}
```

### 2. **监听网络状态变化**（动态注册）

```kotlin
class MainActivity : AppCompatActivity() {
    private lateinit var networkReceiver: BroadcastReceiver
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        
        // 创建广播接收器
        networkReceiver = object : BroadcastReceiver() {
            override fun onReceive(context: Context?, intent: Intent?) {
                val connectivityManager = context?.getSystemService(
                    Context.CONNECTIVITY_SERVICE
                ) as ConnectivityManager
                
                val activeNetwork = connectivityManager.activeNetwork
                val isConnected = activeNetwork != null
                
                if (isConnected) {
                    Toast.makeText(context, "网络已连接", Toast.LENGTH_SHORT).show()
                } else {
                    Toast.makeText(context, "网络已断开", Toast.LENGTH_SHORT).show()
                }
            }
        }
        
        // 注册接收器
        val filter = IntentFilter(ConnectivityManager.CONNECTIVITY_ACTION)
        registerReceiver(networkReceiver, filter)
    }
    
    override fun onDestroy() {
        super.onDestroy()
        // 注销接收器，避免内存泄漏
        unregisterReceiver(networkReceiver)
    }
}
```

对应的 AndroidManifest.xml 权限：

```xml
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
```

### 3. **发送自定义广播**

```kotlin
// 发送方：发送广播
val intent = Intent("com.example.CUSTOM_ACTION")
intent.putExtra("message", "Hello Broadcast")
sendBroadcast(intent)

// 接收方：监听自定义广播
class MyBroadcastReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context?, intent: Intent?) {
        val message = intent?.getStringExtra("message")
        Log.d("BroadcastReceiver", "收到消息: $message")
    }
}

// 在 Activity 中动态注册
val receiver = MyBroadcastReceiver()
val filter = IntentFilter("com.example.CUSTOM_ACTION")
registerReceiver(receiver, filter)
```

## 注册方式对比

|方式|优点|缺点|使用场景|
|---|---|---|---|
|**静态注册**（Manifest）|应用关闭后仍能接收|Android 8.0+ 有限制，电池优化影响|系统事件（重启、插入 SIM 卡）|
|**动态注册**（代码中）|灵活可控，即时注册|应用关闭后无法接收|网络状态、电池电量等实时事件|

---

## 重要注意事项

⚠️ **onReceive() 方法有 10 秒超时限制**

- 不能执行长时操作
- 需要后台任务用 `Service` 或 `WorkManager`

⚠️ **Android 8.0+ (API 26) 限制**

- 大多数系统广播不能通过静态注册接收
- 需要动态注册或使用系统 API 例外

⚠️ **内存泄漏风险**

- 动态注册的接收器必须在 `onDestroy()` 中注销
