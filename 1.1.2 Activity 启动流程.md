# 三、ActivityThread 的作用

很多面试官都会问：

```
ActivityThread是什么？
```

很多人回答：

```
主线程
```

其实：

```
不是线程
```

它是：

```
App进程主线程管理类
```

负责：

- 管理 Activity 生命周期
    
- 管理 Application 生命周期
    
- Handler 消息循环
    
- Binder 回调
    
- Resource 加载
    
- ContentProvider 初始化
    

真正的主线程：

```
main()
```

源码：

```java
public static void main(String[] args) {

    Looper.prepareMainLooper();

    ActivityThread thread = new ActivityThread();

    thread.attach(false);

    Looper.loop();

}
```

所以：

```
ActivityThread
↓

Looper

↓

MessageQueue

↓

Handler(H)

↓

Activity生命周期
```

# 四、为什么 Activity 必须运行在主线程？

原因包括：

1. **UI 框架不是线程安全的**，统一在主线程更新可以避免复杂的同步问题。
    
2. **ViewRootImpl** 会检查线程一致性，跨线程更新 UI 会抛出异常：
    

```java
CalledFromWrongThreadException
```

源码：

```java
void checkThread() {
    if (mThread != Thread.currentThread()) {
        throw new CalledFromWrongThreadException();
    }
}
```

3. 主线程拥有 `Looper`，负责处理生命周期、输入事件、绘制等消息。

# 五、Activity 启动涉及的核心类

|类|作用|
|---|---|
|`Activity`|页面组件，开发者编写的界面逻辑|
|`Instrumentation`|应用侧启动入口，负责发起启动请求和生命周期回调|
|`ActivityTaskManagerService (ATMS)`|系统侧管理 Activity、Task、任务栈和启动流程|
|`ActivityStarter`|处理启动模式、任务栈复用、启动合法性等逻辑|
|`ActivityRecord`|系统中表示一个 Activity 实例的记录对象|
|`ApplicationThread`|应用进程中的 Binder 接口，接收系统回调|
|`ActivityThread`|应用主线程管理类，负责创建 Activity 和执行生命周期|
|`Instrumentation.newActivity()`|通过反射创建 Activity 实例|
|`PhoneWindow`|每个 Activity 对应的窗口对象|
|`WindowManager`|管理窗口的添加、更新和移除|

# 六、高频面试追问

面试官通常会进一步追问：

1. **`startActivity()` 为什么最终要经过 `Instrumentation`？**
    
    - `Instrumentation` 是应用侧与系统启动流程的桥梁，负责统一封装启动请求，并在创建完成后回调生命周期方法。
        
2. **为什么要通过 Binder 调用系统服务？**
    
    - `Activity` 的启动涉及任务栈、窗口、权限和进程管理，这些都属于系统能力，由系统进程中的 `ActivityTaskManagerService` 统一管理，应用不能直接操作。
        
3. **`ActivityRecord` 和 `Activity` 有什么区别？**
    
    - `ActivityRecord` 存在于系统进程，记录 Activity 的状态、任务栈信息和生命周期；`Activity` 对象存在于应用进程，负责界面和业务逻辑。两者通过 Binder Token 建立关联。
        
4. **`ActivityThread` 为什么叫 Thread，却不是线程？**
    
    - 它是运行在主线程中的管理类，本身不是 `java.lang.Thread` 的子类，而是负责调度主线程上的生命周期和消息处理。
        
5. **`attach()` 做了哪些事情？**
    
    - 为 `Activity` 绑定 `Context`、`Application`、`Instrumentation`、`PhoneWindow`、`WindowManager`、Token 等运行环境，为后续 `setContentView()` 和窗口显示做好准备。
        

掌握以上流程，并能够结合 `Instrumentation → ATMS → ActivityThread → performLaunchActivity()` 的调用链讲解，再配合关键源码和设计原因，通常可以满足大多数 Android 中高级岗位关于 Activity 启动机制的面试要求。
