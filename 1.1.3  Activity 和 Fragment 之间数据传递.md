# Activity 和 Fragment 之间数据传递

这是 Android 开发中的核心问题。我为你介绍几种常用方式：

## 1. **Bundle（基础方式）**

### Fragment → Activity

```kotlin
// Fragment 中传递数据给 Activity
val bundle = Bundle().apply {
    putString("key", "value")
    putInt("count", 123)
}

// 通过 intent 启动 Activity
startActivity(Intent(requireContext(), MainActivity::class.java).apply {
    putExtras(bundle)
})
```

### Activity 中接收

```kotlin
val data = intent.getStringExtra("key")
val count = intent.getIntExtra("count", 0)
```

---

## 2. **Bundle + Arguments（推荐的 Fragment 接收方式）**

### Activity 传递给 Fragment

```kotlin
// Activity 中
val fragment = MyFragment().apply {
    arguments = Bundle().apply {
        putString("key", "value")
        putInt("count", 123)
    }
}

supportFragmentManager.beginTransaction()
    .replace(R.id.container, fragment)
    .commit()
```

### Fragment 接收

```kotlin
class MyFragment : Fragment() {
    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)
        val data = arguments?.getString("key")
        val count = arguments?.getInt("count", 0)
    }
}
```

---

## 3. **ViewModel（现代推荐方式 ⭐**）

这是官方推荐的做法，特别适合 Activity 和 Fragment 之间的数据共享：

```kotlin
// 共享 ViewModel
class SharedViewModel : ViewModel() {
    private val _sharedData = MutableLiveData<String>()
    val sharedData: LiveData<String> = _sharedData
    
    fun updateData(newData: String) {
        _sharedData.value = newData
    }
}

// 在 Fragment 中
class MyFragment : Fragment() {
    private val viewModel: SharedViewModel by viewModels()
    
    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)
        
        // 观察数据变化
        viewModel.sharedData.observe(viewLifecycleOwner) { data ->
            // 更新 UI
        }
    }
}

// 在 Activity 中
class MainActivity : AppCompatActivity() {
    private val viewModel: SharedViewModel by viewModels()
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        
        // 修改数据
        viewModel.updateData("new value")
    }
}
```

**为什么推荐 ViewModel？**

- ✅ 数据在配置变化（旋转屏幕）时保留
- ✅ Activity 和 Fragment 之间通过共享 ViewModel 解耦
- ✅ 生命周期安全（不会内存泄漏）
- ✅ 支持 LiveData 和 StateFlow

## 4. **Fragment 和 Activity 通过回调通信**

### Fragment 定义接口

```kotlin
class MyFragment : Fragment() {
    interface OnDataListener {
        fun onDataReceived(data: String)
    }
    
    private lateinit var listener: OnDataListener
    
    override fun onAttach(context: Context) {
        super.onAttach(context)
        listener = context as? OnDataListener 
            ?: throw ClassCastException("Activity must implement OnDataListener")
    }
    
    private fun sendDataToActivity() {
        listener.onDataReceived("fragment data")
    }
}
```

### Activity 实现接口

```kotlin
class MainActivity : AppCompatActivity(), MyFragment.OnDataListener {
    override fun onDataReceived(data: String) {
        // 处理来自 Fragment 的数据
        Log.d("TAG", "Received: $data")
    }
}
```

## 5. **使用 StateFlow（Kotlin Flow，更现代）**

```kotlin
class SharedViewModel : ViewModel() {
    private val _uiState = MutableStateFlow<UiState>(UiState.Idle)
    val uiState: StateFlow<UiState> = _uiState.asStateFlow()
    
    fun updateState(newState: UiState) {
        _uiState.value = newState
    }
}

// Fragment 中收集数据
class MyFragment : Fragment() {
    private val viewModel: SharedViewModel by viewModels()
    
    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)
        
        viewLifecycleOwner.lifecycleScope.launch {
            viewModel.uiState.collect { state ->
                // 响应状态变化
            }
        }
    }
}
```

## 选择指南

|方式|适用场景|优点|缺点|
|---|---|---|---|
|**Bundle**|简单数据传递|简洁，序列化支持|仅支持可序列化数据|
|**ViewModel**|Activity ↔ Fragment|生命周期安全，解耦，配置变化保留|需要依赖|
|**回调**|Fragment → Activity|灵活|易内存泄漏，代码重复|
|**StateFlow**|实时数据流|响应式，现代化|学习曲线|

---

**推荐方案：** 对于大多数场景，**ViewModel + LiveData/StateFlow** 是最佳实践。它是官方推荐的架构模式，能够自动处理生命周期问题，避免内存泄漏。
