# 1. Handler源码分析（必问）

## 面试题

Handler为什么不会阻塞主线程？

---

## 标准答案

Android主线程本质是：

```java
public static void main(String[] args) {
    Looper.prepareMainLooper();
    Looper.loop();
}
```

主线程一直在执行：

```java
Looper.loop()
```

不断从：

```java
MessageQueue
```

取消息。

消息来了处理。

没有消息休眠。

因此不会死循环耗CPU。

---

## 整体架构

```text
Handler
   ↓
Message
   ↓
MessageQueue
   ↓
Looper
   ↓
nativePollOnce()
   ↓
epoll_wait()
```

---

## Looper.prepare()

源码：

```java
public static void prepare() {
    sThreadLocal.set(new Looper());
}
```

创建：

```java
Looper
MessageQueue
```

绑定当前线程。

---

## Handler创建

```java
Handler handler = new Handler();
```

源码：

```java
mLooper = Looper.myLooper();
mQueue = mLooper.mQueue;
```

拿到当前线程Looper。

---

## sendMessage()

```java
handler.sendMessage(msg);
```

源码：

```java
enqueueMessage()
```

进入：

```java
MessageQueue
```

---

## Looper.loop()

源码：

```java
for (;;) {
    Message msg = queue.next();

    msg.target.dispatchMessage(msg);
}
```

无限循环。

---

## queue.next()

Java层：

```java
nativePollOnce(ptr, timeout);
```

进入JNI。

---

## nativePollOnce()

进入：

```cpp
android_os_MessageQueue.cpp
```

调用：

```cpp
Looper::pollOnce()
```

---

## 最终调用

Linux：

```cpp
epoll_wait()
```

等待事件。

---

## 加分回答

为什么不卡CPU？

因为：

```cpp
epoll_wait()
```

是阻塞等待。

没有消息：

```text
睡眠
```

有消息：

```text
唤醒
```

CPU几乎不消耗。

# 2. Binder IPC源码分析（必问）

---

## 面试题

为什么Android不用Socket做IPC？

---

## 标准答案

Binder优势：

```text
效率高
安全
支持权限控制
一次内存拷贝
```

Socket：

```text
两次拷贝
无权限管理
```

---

## Binder架构

```text
Client
 ↓
Proxy
 ↓
Binder Driver
 ↓
Stub
 ↓
Service
```

---

## AIDL生成代码

例如：

```java
interface IUserService
```

生成：

```java
Stub
Proxy
```

---

## 调用流程

```java
userService.getUser();
```

实际：

```java
Proxy.transact()
```

↓

```cpp
ioctl()
```

↓

```text
Binder Driver
```

↓

```java
Stub.onTransact()
```

↓

```java
真实Service
```

---

## Binder为什么快

共享内存：

```text
mmap
```

映射：

```text
用户空间
↓
内核空间
↓
用户空间
```

只拷贝一次。

# Android高级面试必背框架图

```text
Android Framework
│
├── Activity启动流程
│      └── AMS
│
├── View显示流程
│      └── WMS
│
├── APK安装流程
│      └── PMS
│
├── IPC通信
│      └── Binder
│
├── 消息机制
│      └── Handler
│
├── 网络
│      └── OkHttp
│
├── 数据库
│      └── Room
│
├── 多线程
│      └── Coroutine
│
└── 性能优化
       ├── OOM
       ├── ANR
       └── 启动优化
```

按照这份内容准备，你面试 Android 3~5 年经验岗位时，面试官追问到 Framework 源码层（AMS/WMS/PMS、Binder Driver、epoll、Coroutine 状态机）基本都能接住。最关键的是把 **Handler → Binder → Activity启动 → View绘制 → RecyclerView → OkHttp → Coroutine** 这条主线讲顺，这是中高级 Android 面试命中率最高的一组题。
