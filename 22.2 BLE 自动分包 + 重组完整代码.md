# 一、BLE为什么要分包

BLE单包长度：

```text
ATT_MTU = 23

23 - 3 = 20字节有效数据
```

默认：

```text
1包 = 20Byte
```

如果发送：

```text
100Byte
```

实际：

```text
20
20
20
20
20
```

拆成：

```text
5包
```

# 二、协议设计

工业项目推荐协议：

```text
┌────┬────┬──────┬──────┬────────┬──────┐
│AA55│CMD │LEN   │SEQ   │DATA    │CRC16 │
└────┴────┴──────┴──────┴────────┴──────┘
```

例如：

```text
AA55
01
0064
01
100字节数据
CRC16
```

字段：

|字段|长度|
|---|---|
|Header|2|
|CMD|1|
|LEN|2|
|SEQ|1|
|DATA|N|
|CRC|2|

# 三、发送自动分包

## BlePacketUtil

```java
public class BlePacketUtil {

    private static final int PACKET_SIZE = 20;

    public static List<byte[]> split(byte[] data) {

        List<byte[]> packets = new ArrayList<>();

        int length = data.length;

        int count = (int) Math.ceil(length * 1.0 / PACKET_SIZE);

        for (int i = 0; i < count; i++) {

            int start = i * PACKET_SIZE;

            int end = Math.min(start + PACKET_SIZE, length);

            packets.add(
                    Arrays.copyOfRange(data, start, end)
            );
        }

        return packets;
    }
}
```

---

## 使用

```java
byte[] command = buildCommand();

List<byte[]> packets =
        BlePacketUtil.split(command);
```

结果：

```text

```

```java
public void sendData(byte[] data) {

    List<byte[]> packets =
            BlePacketUtil.split(data);

    Observable.fromIterable(packets)

                    bleConnection.writeCharacteristic(
                            writeUUID,
                            packet
                    ).toObservable()

                    .delay(30, TimeUnit.MILLISECONDS)

            )

            .subscribe(
                    bytes -> {

                    },
                    Throwable::printStackTrace
            );
}
```

特点：

```text

```

# 五、支持MTU动态分包

连接成功：

```java
bleDevice.establishConnection(false)

        .flatMap(rxBleConnection ->

                rxBleConnection.requestMtu(247)

        )
        .subscribe(mtu -> {

            Log.d("BLE","MTU="+mtu);

        });
```

---

动态计算包大小：

```java
int payloadSize = mtu - 3;
```

例如：

```text
247 - 3 = 244
```

即：

```text
1包可发244字节
```

---

修改工具类：

```java
public static List<byte[]> split(
        byte[] data,
        int mtu
) {

    int packetSize = mtu - 3;

    List<byte[]> list = new ArrayList<>();

    for(int i=0;i<data.length;i+=packetSize){

        int end =
                Math.min(
                        i + packetSize,
                        data.length
                );

        list.add(
                Arrays.copyOfRange(
                        data,
                        i,
                        end
                )
        );
    }

    return list;
}
```

# 六、接收自动重组

很多设备返回：

```text
20
20
20
10
```

需要拼接。

---

## BleAssembler

```java
public class BleAssembler {

    private final ByteArrayOutputStream buffer =
            new ByteArrayOutputStream();

    public byte[] append(byte[] packet) {

        try {

            buffer.write(packet);

            byte[] all =
                    buffer.toByteArray();

            if(isComplete(all)){

                buffer.reset();

                return all;
            }

        } catch (Exception e) {
            e.printStackTrace();
        }

        return null;
    }

    private boolean isComplete(byte[] data){

        if(data.length < 6){
            return false;
        }

        int len =
                ((data[3] & 0xFF) << 8)
                        | (data[4] & 0xFF);

        return data.length >= len + 8;
    }
}
```

# 九、工业级方案（推荐）

实际项目不要仅靠长度判断，建议增加包序号和结束标记。

```text
AA55
CMD
TOTAL_LEN
PACKET_NO
TOTAL_PACKET
DATA
CRC16
```

例如：

```text
AA55
01
100
01
05
...
```

表示：

```text
总长度=100
当前第1包
总共5包
```

接收端：

```java
Map<Integer, byte[]> packetMap
```

按包号缓存：

```text
1
2
3
4
5
```

全部收到后：

```java
merge()
```

这样即使丢包、乱序，也能正确重组。

### 面试高频回答

当面试官问“BLE长数据怎么处理”时，可以回答：

> BLE默认MTU为23字节，实际有效载荷只有20字节。发送大数据时需要根据MTU自动分包，使用队列或concatMap顺序发送；接收端根据协议头、总长度、包序号进行缓存和重组，待完整数据到达后再解析。工业项目中通常会增加CRC校验、ACK确认、超时重发和分包序号机制来保证传输可靠性。
