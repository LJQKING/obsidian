**MVVM 生命周期**（Model-View-ViewModel Lifecycle）主要指的是 **ViewModel** 在整个应用运行过程中的创建、使用和销毁过程。MVVM 本身是一种架构模式，并没有像 Android Activity 那样严格的系统级生命周期，但不同平台/框架对 ViewModel 提供了不同的生命周期管理机制。

### 1. MVVM 核心组件生命周期概述

| 组件     | 生命周期关键点                          | 典型行为 |
|----------|---------------------------------------|----------|
| **View** (页面/Fragment/Activity) | 创建 → 可见 → 交互 → 销毁 | 负责绑定 ViewModel，处理 UI 事件 |
| **ViewModel** | 创建 → 活跃 → 清理（onCleared）→ 销毁 | 持有业务逻辑和状态，独立于 View 配置变更 |
| **Model**    | 由 ViewModel 按需创建/销毁             | 数据层，通常无独立生命周期 |

---

### 2. Android Jetpack MVVM 生命周期（最常用）

Google 官方推荐的 Jetpack MVVM 中，**ViewModel** 是生命周期感知的：

#### ViewModel 生命周期
1. **创建**（`ViewModelProvider` 第一次获取时）
   - 通常在 `onCreate()` 或 Fragment 的 `onCreateView()` 中通过 `ViewModelProvider` 获取。
   - 如果是 Activity/Fragment 重建（旋转屏幕等），ViewModel **不会重新创建**，会复用之前的实例（数据得以保留）。

2. **活跃期**
   - ViewModel 一直存活，直到关联的 **Activity/Fragment 被永久销毁**。
   - 可使用 `LiveData`、`StateFlow`、`SharedFlow` 等观察数据变化。

3. **清理**（`onCleared()`）
   - 当 Activity/Fragment **永久销毁**（不是配置变更）时调用。
   - 适合释放资源：取消网络请求、关闭数据库连接、清理 RxJava/协程等。

**示例代码**（Kotlin）：
```kotlin
class MyViewModel : ViewModel() {
    
    override fun onCleared() {
        super.onCleared()
        // 清理工作
        repository.cancelAllJobs()
        compositeDisposable.clear()
    }
}
```

**Fragment + ViewModel 生命周期对应关系**：
- Fragment `onCreate()` → ViewModel 创建（或复用）
- Fragment `onViewCreated()` → 绑定 UI + 观察 LiveData/Flow
- Fragment `onDestroy()`（配置变更时）→ ViewModel **继续存活**
- Fragment `onDestroy()`（永久销毁）→ ViewModel `onCleared()`

### 3. WPF / .NET MVVM 生命周期

- **View**（Window / UserControl）：Loaded → Unloaded
- **ViewModel**：
  - 通常在 View 的 DataContext 中实例化，或通过 IoC 容器注入。
  - 可实现 `IDisposable` 接口，在 View Unloaded 时释放资源。
  - 常用 `INotifyPropertyChanged` + 命令模式。

常见做法：
```csharp
public class MyViewModel : INotifyPropertyChanged, IDisposable
{
    public void Dispose() { /* 释放资源 */ }
}
```

### 4. 其他平台

- **iOS (SwiftUI / Combine)**：使用 `@StateObject` 或 `ObservableObject`，在 View 消失时自动释放。
- **Flutter (Provider / Riverpod / Bloc)**：Provider 有 `dispose()`，Bloc 有 `close()`。
- **Vue.js (Composition API)**：`setup()` → `onBeforeUnmount` / `onUnmounted`。

---

### 5. 最佳实践建议

1. **ViewModel 应保持轻量且无 View 引用**（不要持有 Activity/Fragment 实例）。
2. **数据持久化**：使用 `SavedStateHandle`（Android）或类似机制应对进程被杀。
3. **资源释放**一定要在 `onCleared()` / `dispose()` 中处理，避免内存泄漏。
4. **使用 Lifecycle-Aware 组件**（LiveData、ViewModel、LifecycleOwner）自动感知生命周期。
5. **测试友好**：ViewModel 不依赖 Android Context，便于单元测试。
