# Android 插件化开发详解

插件化（Plugin Architecture）是指：

> 不安装 APK 到系统中，而是在宿主 App（Host）运行时动态加载另一个 APK（Plugin）中的 Activity、Service、资源和代码。

典型框架：

- [RePlugin](https://github.com/Qihoo360/RePlugin?utm_source=chatgpt.com)
    
- [DroidPlugin](https://github.com/DroidPluginTeam/DroidPlugin?utm_source=chatgpt.com)
    
- [VirtualAPK](https://github.com/didi/VirtualAPK?utm_source=chatgpt.com)
    
- [Shadow](https://github.com/Tencent/Shadow?utm_source=chatgpt.com)
    

---

# 一、为什么需要插件化

例如：

今日头条

主APK：

```
Host.apk
```

插件：

```
News.apk
Video.apk
Mall.apk
```

运行时：

```
Host
 ├── News Plugin
 ├── Video Plugin
 └── Mall Plugin
```

优点：

- 动态更新模块
    
- 减少主包体积
    
- 功能按需下载
    
- 热修复基础能力

```java
DexPath
优化目录
so目录
父ClassLoader
```

# 加载插件类

插件：

```java
public class Test {
    public void hello() {
        Log.d("Plugin", "hello");
    }
}
```

宿主：

```java
Class<?> clazz =
        classLoader.loadClass(
                "com.plugin.Test");

Object obj = clazz.newInstance();

Method method =
        clazz.getMethod("hello");

method.invoke(obj);
```

运行结果：

```java
hello
```

# 四、资源加载原理

正常：

```java
context.getResources()
```

只能访问宿主资源。

插件：

```java
plugin.apk
```

资源无法找到。

---

## AssetManager加载插件资源

源码：

```java
AssetManager assetManager =
        AssetManager.class.newInstance();

Method addAssetPath =
        AssetManager.class.getMethod(
                "addAssetPath",
                String.class);

addAssetPath.invoke(
        assetManager,
        pluginPath);
```

---

构造Resources

```java
Resources pluginRes =
        new Resources(
                assetManager,
                superRes.getDisplayMetrics(),
                superRes.getConfiguration());
```

---

获取资源

```java
int id = pluginRes.getIdentifier(
        "plugin_text",
        "string",
        "com.plugin");

String text =
        pluginRes.getString(id);
```

# 六、代理Activity方案

最经典方案。

宿主注册：

```xml
<activity
    android:name=".ProxyActivity"/>
```

AMS认为：

```java
ProxyActivity
```

合法。

---

实际启动：

```java
Intent intent =
        new Intent();

intent.putExtra(
        "className",
        "com.plugin.MainActivity");

startActivity(intent);
```

---

ProxyActivity：

```java
public class ProxyActivity
        extends Activity {

    @Override
    protected void onCreate(
            Bundle savedInstanceState) {

        super.onCreate(savedInstanceState);

        String className =
                getIntent()
                        .getStringExtra(
                                "className");

        launchPlugin(className);
    }
}
```

# 七、反射创建插件Activity

```java
Class<?> clazz =
        pluginClassLoader
                .loadClass(className);

Object instance =
        clazz.newInstance();
```

---

插件Activity统一接口：

```java
public interface IPluginActivity {

    void attach(Activity proxy);

    void onCreate(Bundle bundle);

}
```

---

插件实现：

```java
public class PluginActivity
        implements IPluginActivity {

    Activity proxy;

    @Override
    public void attach(
            Activity proxy) {

        this.proxy = proxy;
    }

    @Override
    public void onCreate(
            Bundle bundle) {

        TextView tv =
                new TextView(proxy);

        tv.setText("插件页面");

        proxy.setContentView(tv);
    }
}
```

---

ProxyActivity

```java
IPluginActivity plugin =
        (IPluginActivity)
        clazz.newInstance();

plugin.attach(this);

plugin.onCreate(new Bundle());
```

结果：

```
ProxyActivity
      ↓
PluginActivity
```

用户看到：

```
PluginActivity
```

实际上系统启动的是：

```
ProxyActivity
```

# 九、Hook Instrumentation源码

获取ActivityThread：

```java
Class<?> clazz =
        Class.forName(
            "android.app.ActivityThread");

Method method =
        clazz.getDeclaredMethod(
            "currentActivityThread");

Object activityThread =
        method.invoke(null);
```

---

获取Instrumentation：

```java
Field field =
        clazz.getDeclaredField(
            "mInstrumentation");

field.setAccessible(true);

Instrumentation base =
        (Instrumentation)
        field.get(activityThread);
```

---

替换：

```java
field.set(
        activityThread,
        new HookInstrumentation(base));
```

---

Hook类：

```java
public class HookInstrumentation
        extends Instrumentation {

    Instrumentation base;

    public HookInstrumentation(
            Instrumentation base) {

        this.base = base;
    }

    @Override
    public Activity newActivity(
            ClassLoader cl,
            String className,
            Intent intent)
            throws Exception {

        return super.newActivity(
                pluginClassLoader,
                pluginRealClass,
                intent);
    }
}
```

# 十、插件化ClassLoader原理

Android默认：

```text
BootClassLoader
        ↑
PathClassLoader
```

插件化：

```text
BootClassLoader
        ↑
PathClassLoader
        ↑
DexClassLoader
```

---

加载顺序：

```java
loadClass()
```

源码：

```java
parent.loadClass()
```

先父后子。

---

插件框架改造：

```java
PluginClassLoader
```

实现：

```java
@Override
protected Class<?> loadClass(
        String name,
        boolean resolve)
```

优先插件。

```text
Plugin First
```

