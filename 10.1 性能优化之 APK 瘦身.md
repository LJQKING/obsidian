# 一、APK组成结构

APK本质：

```bash
xxx.apk
```

其实就是：

```bash
zip压缩包
```

解压后：

```bash
app.apk
│
├── AndroidManifest.xml
├── classes.dex
├── classes2.dex
├── resources.arsc
├── assets/
├── res/
├── lib/
│    ├── armeabi-v7a
│    ├── arm64-v8a
│    └── x86
└── META-INF/
```

查看APK：

```bash
unzip app.apk
```

或者：

```bash
apkanalyzer apk summary app.apk
```

# 2 Tree Shaking

原理：

```text
从Application开始

向下查找

构建引用树

未被引用的删除
```

例如：

```java
class A {
}

class B {
}
```

只用了A：

```java
new A();
```

R8：

```java
class B
```

直接删除。

---

源码：

```java
TreePruner.java
```

```java
public DexApplication run() {

    for (DexProgramClass clazz :
         application.classes()) {

        if (!liveTypes.contains(clazz.type)) {

            unused.remove(clazz);
        }
    }

}
```

意思：

```text
不在引用树中的类

直接删除
```

# 十、面试标准答案（3分钟）

> APK瘦身本质是减少Dex、资源、图片和SO库体积。
> 
> Dex层面主要通过R8实现Tree Shaking、方法内联、常量折叠等优化，其核心源码在R8的TreePruner和InliningOracle中。
> 
> 资源层面通过shrinkResources分析R引用关系，删除未使用资源，核心实现是AGP中的ResourceUsageAnalyzer。
> 
> 图片层面主要将PNG转换WebP，压缩率通常能达到30%-70%。
> 
> Native层面通过abiFilters只保留armeabi-v7a和arm64-v8a架构。
> 
> 如果发布到Google Play，则使用App Bundle动态生成Split APK，按设备ABI、语言和屏幕密度裁剪资源，可进一步减少30%-50%的安装包体积。
> 
> 实际项目中，结合R8、资源压缩、WebP、SO裁剪和动态化加载，通常可以把APK体积降低40%以上。
