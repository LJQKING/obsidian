# Android 开发者知识库 — 知识类型分析与序列排序分类

## 功能说明

对 Obsidian 知识库中全部 110 个 Markdown 文件进行知识类型识别、聚类分组与学习序列排序，形成结构化的知识地图。

## 已完成内容

- 统计：共 110 个 `.md` 文件，约 6 万行笔记
- 识别出 10 大知识类别
- 每个类别内部按"基础→进阶→实战"三级排序
- 标注了每篇笔记的文件名、行数、知识点标签

## 涉及文件

- 知识库根目录下全部 `.md` 文件（110 个）

## 测试方法

- 手动抽查 20 个文件的分类准确性
- 验证每个文件只属于一个主类别
- 检查层级排序是否符合学习递进关系

## 已知问题

- 部分文件存在重叠（如 32.compose 与 32.0 Compose），实际内容可能重复
- 文件编号（1.x、2.x）反映作者原始组织方式，与本分类体系的序号不完全对应
- 行数统计基于 wc -l，实际有效代码/文本行数略少

## 后续优化

- 可为每个知识点添加"已掌握/学习中/未学习"状态标记
- 可生成可视化知识图谱（Mermaid 或 D3.js）
- 可基于此分类自动生成学习计划（每日阅读清单）
- 建议将重复文件合并或建立索引链接

## 注意事项

- 本分类以"学习路径"为主线，而非单纯按文件编号排列
- 难度等级为初级/中级/高级，为主观评估，仅供参考
- 车载专项（BLE/MQTT/商米SDK）适合有车载开发需求的读者优先阅读
