## Binder为什么比Socket快？

Socket：

```text
用户空间
↓

内核空间
↓

用户空间
```

```text
2次数据拷贝
4次上下文切换
```

---

Binder：

利用：

```text
mmap
```

共享内存。

```text
1次数据拷贝
```

性能更高。

## Looper为什么死循环不卡CPU？

源码：

```java
for (;;) {
   Message msg = queue.next();
}
```

核心：

```java
nativePollOnce()
```

进入Linux：

```cpp
epoll_wait()
```

没有消息：

```text
线程休眠
CPU≈0
```

有消息：

```text
epoll唤醒
```

---

## MessageQueue为什么是单链表？

源码：

```java
Message mMessages;
```

特点：

```text
插入O(n)
删除O(1)
```

消息量通常较少。

---

## CAN仲裁机制

规则：

```text
0 = 显性位
1 = 隐性位
```

显性位覆盖隐性位。

例如：

```text
0x100
0x200
```

结果：

```text
0x100优先
```

因为：

```text
ID越小
优先级越高
```

# 面试优先级（建议学习顺序）

```text
★★★★★ Binder源码
★★★★★ Handler + Looper + epoll
★★★★★ AMS/WMS/PMS
★★★★★ AAOS(CarService)
★★★★★ Vehicle HAL
★★★★★ CAN总线

★★★★ RecyclerView
★★★★ JNI/NDK
★★★★ Linux
```

对于你目前已有的 Android 基础（Java、Kotlin、MVVM、Room、Socket、BLE、RecyclerView、Service），补齐 **Binder → Framework → Linux → AAOS → Vehicle HAL → CAN** 这条链路后，就已经接近大多数车企（如 BYD、NIO、XPeng、Li Auto）中高级 Android 车载开发岗位的核心面试要求。
